151. Reverse Words in a String

Reverse the order of words in a string while collapsing multiple spaces into single ones and stripping leading/trailing whitespace. The key is: split cleanly, then walk the token list from the right and rebuild.

Microsoft · Medium

Summary

Hint: trim() the input, split(" ") on spaces, then iterate the token array from right to left. Skip any empty tokens (that's how you absorb consecutive spaces), appending the non-empty ones into a StringBuilder with a single separator. Final trim() removes the trailing space.

Answer

✅ Split, walk right→left, skip empties

✅ Solution.java — split & rebuild 
class Solution {
    public String reverseWords(String s) {
        String[] words = s.trim().split(" ");

        StringBuilder sb = new StringBuilder();
        for(int i = words.length - 1; i >= 0; i--) {
            if(words[i].trim().isEmpty() == false) {
                sb.append(words[i]);
                sb.append(" ");
            }
        }
        return sb.toString().trim();
    }
}
Why the empty-check? "a b".split(" ") yields ["a","","","b"] — the extra spaces produce empty tokens. Skipping them is what collapses runs of whitespace into a single separator in the output.

Alternative one-liners (good to mention)

  • split("\\s+") regex collapses multi-space runs, saving the inner empty check:
    return String.join(" ", new StringBuilder(String.join(" ", s.trim().split("\\s+"))).reverse()…) — elegant but relies on regex + reverse tricks.
  • In-place O(1) extra space (interview stretch): reverse the whole char[], then reverse each word, then compact the spaces with two pointers.

Complexity

  • Time: O(n) — single pass to split plus one pass over the tokens.
  • Space: O(n) — the token array and the StringBuilder for the result.
