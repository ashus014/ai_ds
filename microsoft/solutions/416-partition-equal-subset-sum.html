416. Partition Equal Subset Sum

Given a non-empty array nums of positive integers, return true if you can partition the elements into two subsets with equal sum. Equivalently: is there a subset whose sum equals half of the total sum of nums?

Microsoft · Medium

Summary

Hint: If the total sum is odd, splitting into two equal halves is impossible. If it is even, let target = sum / 2. The problem becomes the classic 0/1 knapsack decision: can we pick a subset of items (each used at most once) whose weights sum to exactly target? Try every item with DFS (take vs skip); memoize on (index, remaining) because the same subproblem is reached many times.

Answers

Naive — exponential DFS (take / skip)

At each index, branch: either put nums[index] in the first subset (subtract from target) or leave it out. Base cases: target == 0 (built the half-sum), target < 0 or past the end of the array (failure).

Solution.java — naive DFS 
class Solution {

    public boolean canPartition(int[] nums) {
        int sum = 0;

        for (int x : nums) sum += x;

        // if total sum is odd → cannot split equally
        if (sum % 2 != 0) return false;

        int target = sum / 2;

        return dfs(nums, 0, target);
    }

    boolean dfs(int[] nums, int index, int target) {

        // 🎯 found subset
        if (target == 0) return true;

        // ❌ out of bounds or exceeded
        if (index == nums.length || target < 0) return false;

        // take OR not take
        return dfs(nums, index + 1, target - nums[index]) 
            || dfs(nums, index + 1, target);
    }
}
Correct logic, but the recursion tree explodes: roughly 2n paths in the worst case. LeetCode will time out for larger n.

Optimal — DFS + memoization

State is fully described by (index, target): "starting from index, can we still reach sum target?" Cache each result in dp[index][target] (null = unknown, true/false = computed). Same recursion as naive, but each state is evaluated once.

Solution.java — memoized DFS 
class Solution {

    Boolean[][] dp;

    public boolean canPartition(int[] nums) {
        int sum = 0;

        for (int x : nums) sum += x;

        if (sum % 2 != 0) 
            return false;

        int target = sum / 2;

        dp = new Boolean[nums.length][target + 1];

        return dfs(nums, 0, target);
    }

    boolean dfs(int[] nums, int index, int target) {

        if (target == 0) 
            return true;

        if (index == nums.length || target < 0) 
            return false;

        if (dp[index][target] != null){
            return dp[index][target];
        } 

        return dp[index][target] =
            dfs(nums, index + 1, target - nums[index]) ||
            dfs(nums, index + 1, target);
    }
}
Why Boolean wrapper. We need three states for each cell: "not computed yet" (null), true, and false. A primitive boolean[][] cannot represent "unknown" separately from false.

Walkthrough — nums = [1, 5, 11, 5]

Sum = 22, even → target = 11. One valid split is {1, 5, 5} vs {11}, both sum to 11 → return true. The memoized DFS explores take/skip branches but never recomputes the same (index, target) pair twice.

Edge cases

  • Odd total sum. Immediate false — two integers cannot both equal sum/2.
  • Single element. Cannot form two non-empty partitions in the spirit of "two subsets," but for n = 1 and value 0 edge cases are rare; typical constraint sizes still work because target = sum/2 with one positive number means you need that one number to equal half of itself, impossible unless 0.
  • Large numbers, small n. Memo table is O(n · target); if target is huge, consider bitset DP or meet-in-the-middle for very small n.

Complexity

Approach Time Space
Naive DFS O(2n) (exponential) O(n) recursion stack
DFS + memo O(n · target) states, each O(1) O(n · target) for dp + O(n) stack
Common follow-up — bottom-up 1D DP. A boolean array dp[s] meaning "sum s achievable" can be updated in descending s for each number, using O(target) extra space instead of the full 2D table.
