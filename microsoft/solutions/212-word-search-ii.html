212. Word Search II

Generalization of 79. Word Search — given a board and a list of words, return every word that can be formed by walking 4-connected paths without reusing a cell. The trick: instead of running the single-word DFS once per word, walk the board once and let a Trie guide the search across all words at the same time.

Microsoft · Hard

Summary

Hint: Build a Trie of all words. Start a DFS from every cell; at each step, descend into the Trie child for board[i][j] — if the child is missing, prune. When you land on a node where isEndOfWord is true, push node.word to the result and set isEndOfWord = false to avoid duplicates. Mark the cell visited (write '$') before recursing into 4 neighbors, then restore on the way back.

Answers

Naive — run LC 79 per word

Solution.java — per-word DFS 
class Solution {

    int m, n;
    int[][] directions = {{1,0}, {-1,0}, {0,1}, {0,-1}};
    List<String> result = new ArrayList<>();

    public List<String> findWords(char[][] board, String[] words) {

        m = board.length;
        n = board[0].length;

        for (String word : words) {

            if (exist(board, word)) {
                result.add(word);
            }
        }

        return result;
    }

    public boolean exist(char[][] board, String word) {

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {

                if (board[i][j] == word.charAt(0) && find(board, i, j, 0, word)) {
                    return true;
                }
            }
        }

        return false;
    }

    public boolean find(char[][] board, int i, int j, int idx, String word) {

        if (idx == word.length()) {
            return true;
        }

        if (i < 0 || j < 0 || i >= m || j >= n || board[i][j] == '$') {
            return false;
        }

        if (board[i][j] != word.charAt(idx)) {
            return false;
        }

        char temp = board[i][j];
        board[i][j] = '$';

        for (int[] dir : directions) {
            int new_i = i + dir[0];
            int new_j = j + dir[1];

            if (find(board, new_i, new_j, idx + 1, word)) {
                board[i][j] = temp;
                return true;
            }
        }

        board[i][j] = temp;

        return false;
    }
}
Why this TLEs at scale. For each word you re-run the entire LC 79 search across the whole board. With W words of average length L on an m × n board, the work is roughly O(W · m · n · 4^L). The Trie version below shares prefixes across all words and prunes the moment a path stops matching any word.

✅ Trie + DFS — walk the board once

✅ Solution.java — Trie + DFS 
import java.util.*;

class Solution {

    int rows, cols;
    List<String> result = new ArrayList<>();

    public List<String> findWords(char[][] board, String[] words) {

        rows = board.length;
        cols = board[0].length;

        TrieNode root = buildTrie(words);

        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                dfs(board, i, j, root);
            }
        }

        return result;
    }

    private void dfs(char[][] board, int i, int j, TrieNode node) {

        // boundary + visited
        if (i < 0 || j < 0 || i >= rows || j >= cols || board[i][j] == '$') {
            return;
        }

        char ch = board[i][j];

        // no prefix → stop early
        if (node.children[ch - 'a'] == null) return;

        node = node.children[ch - 'a'];

        // word found
        if (node.isEndOfWord) {
            result.add(node.word);
            node.isEndOfWord = false; // avoid duplicates
        }

        char temp = board[i][j];
        board[i][j] = '$';

        dfs(board, i + 1, j, node);
        dfs(board, i - 1, j, node);
        dfs(board, i, j + 1, node);
        dfs(board, i, j - 1, node);

        board[i][j] = temp; // backtrack
    }

    private TrieNode buildTrie(String[] words) {

        TrieNode root = new TrieNode();

        for (String word : words) {

            TrieNode node = root;

            for (char ch : word.toCharArray()) {
                if (node.children[ch - 'a'] == null) {
                    node.children[ch - 'a'] = new TrieNode();
                }
                node = node.children[ch - 'a'];
            }

            node.isEndOfWord = true;
            node.word = word;
        }

        return root;
    }

    class TrieNode {
        boolean isEndOfWord;
        String word;
        TrieNode[] children;

        TrieNode() {
            this.isEndOfWord = false;
            this.word = null;
            this.children = new TrieNode[26];
        }
    }
}
Why store node.word at the end node. When DFS reaches a terminal Trie node, you do not need to reconstruct the path string — the full word is sitting on that node. Setting isEndOfWord = false after collecting it prevents the same word from being added again from a different starting cell. Two common further optimizations: prune empty children from the Trie when a word is consumed, and short-circuit when the result count equals words.length.

Complexity

  • Time (Trie + DFS): build is O(Σ|word|); the search is bounded by O(m · n · 4 · 3^(L−1)) where L is the longest word — every cell starts a DFS that can only descend while the current Trie path exists, so practical runs are far smaller than the naive bound.
  • Time (naive): O(W · m · n · 4^L) — repeats the LC 79 search per word.
  • Space: O(Σ|word|) for the Trie plus O(L) recursion depth; the board is mutated in place and restored.
