17. Letter Combinations of a Phone Number

Given a string of digits 2–9, return every possible letter combination using the classic phone keypad mapping. Backtracking over each digit position produces all 3^k · 4^j words.

Microsoft · Medium — 14 April 2026

Summary

Hint: DFS through the digits string. At each index, append one character from that digit’s letter set, recurse, then backtrack by removing the last character. Collect a result whenever index == digits.length().

Answers

✅ Backtracking

✅ Solution.java — backtracking 
class Solution {

    // Collected letter combinations across the DFS tree
    List<String> result = new ArrayList<>();

    public List<String> letterCombinations(String digits) {
        // Edge case: no digits → no combinations
        if (digits.length() == 0) return new ArrayList<>();

        int index = 0;

        // Phone keypad mapping (digit → letters)
        Map<Character, String> mp = new HashMap<>();
        mp.put('2', "abc");
        mp.put('3', "def");
        mp.put('4', "ghi");
        mp.put('5', "jkl");
        mp.put('6', "mno");
        mp.put('7', "pqrs");
        mp.put('8', "tuv");
        mp.put('9', "wxyz");

        // Current partial word built during DFS
        String st = "";
        helper(index, digits, st, mp);
        return result;
    }

    private void helper(int index, String digits, String st, Map<Character, String> mp) {

        // Base case: placed a letter for every digit → record it
        if (index >= digits.length()) {
            result.add(st);
            return;
        }

        // Look up letters for current digit
        char ch = digits.charAt(index);
        String str = mp.get(ch);

        // Try each letter choice, recurse, then undo the choice
        for (int i = 0; i < str.length(); i++) {
            st += str.charAt(i);                          // pick
            helper(index + 1, digits, st, mp);             // explore
            st = st.substring(0, st.length() - 1);         // unpick (backtrack)
        }
    }
}
Why this works. Each digit contributes one character to the word. Recursion explores the full tree of |letters(d₁)| × |letters(d₂)| × … branches; the backtrack line keeps the partial string consistent with the current path.

Complexity

  • Time: O(4^n · n) — up to 4^n leaves (digit 7/9 has 4 letters), O(n) to build each word.
  • Space: O(n) recursion depth plus the output list. Using a StringBuilder instead of String concatenation trims allocations.
