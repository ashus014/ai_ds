43. Multiply Strings

Multiply two non-negative integers given as strings — without using any built-in bigint library. Simulate grade-school long multiplication: every digit pair num1[i] × num2[j] contributes to positions i+j (carry) and i+j+1 (ones) of the result array.

Microsoft · Medium — 14 April 2026

Summary

Hint: The product of an m-digit and an n-digit number has at most m + n digits. Allocate an int[m+n] result buffer. For every pair (i, j), the digit product plus whatever's already stored at i+j+1 gives sum; write sum % 10 back to i+j+1 and add sum / 10 into i+j (the carry slot). At the end, skip leading zeros.

Answer

✅ Grade-school multiplication with an int[m+n] buffer

✅ Solution.java — long multiplication 
class Solution {
    public String multiply(String num1, String num2) {
        // Either number is "0" → product is "0"
        if (num1.equals("0") || num2.equals("0")) return "0";

        int m = num1.length(), n = num2.length();

        // Product has at most m + n digits (e.g. 99 * 99 = 9801, 2+2 digits)
        int[] res = new int[m + n];

        // Walk both numbers right-to-left (least significant first)
        for (int i = m - 1; i >= 0; i--) {
            for (int j = n - 1; j >= 0; j--) {

                // Multiply the current digit pair
                int mul = (num1.charAt(i) - '0') * (num2.charAt(j) - '0');

                // Add whatever was already buffered at the "ones" slot
                // for this digit pair (previous carry from the cell above)
                int sum = mul + res[i + j + 1];

                // The ones digit stays at i+j+1
                res[i + j + 1] = sum % 10;

                // The tens digit (carry) is added into i+j
                // so it can combine with the NEXT digit-pair's product
                res[i + j] += sum / 10;
            }
        }

        // Convert buffer to a string, skipping leading zeros
        StringBuilder sb = new StringBuilder();
        for (int num : res) {
            // Don't append leading zeros, but do append zeros that follow a non-zero
            if (!(sb.length() == 0 && num == 0)) {
                sb.append(num);
            }
        }

        // If everything was zero (shouldn't happen after the early-exit above,
        // but keeps the function safe), return "0"
        return sb.length() == 0 ? "0" : sb.toString();
    }
}
Why the i+j / i+j+1 positions? In grade-school multiplication, multiplying digit at position i of num1 by digit at position j of num2 (counted from the left) contributes to position i + j (carry column) and i + j + 1 (ones column) of the final result. Writing the ones digit into i+j+1 and adding the carry into i+j correctly distributes partial products without a separate "add partial results" pass.

Dry run — "12" × "34"

  • Initial res = [0,0,0,0].
  • i=1, j=1: 2*4 = 8. sum = 8 + res[3] = 8 → res[3] = 8, res[2] += 0. Buffer: [0,0,0,8].
  • i=1, j=0: 2*3 = 6. sum = 6 + res[2] = 6 → res[2] = 6, res[1] += 0. Buffer: [0,0,6,8].
  • i=0, j=1: 1*4 = 4. sum = 4 + res[2] = 10 → res[2] = 0, res[1] += 1. Buffer: [0,1,0,8].
  • i=0, j=0: 1*3 = 3. sum = 3 + res[1] = 4 → res[1] = 4, res[0] += 0. Buffer: [0,4,0,8].
  • Skip leading zero → "408". ✅ (12 × 34 = 408)

Complexity

  • Time: O(m · n) — every digit pair is processed exactly once.
  • Space: O(m + n) — the result buffer and the output string.
  • Follow-ups. Karatsuba multiplication takes O(nlog₂3) ≈ O(n1.585); Toom–Cook and FFT-based multiplication hit O(n log n log log n) but are overkill for interview input sizes (≤ 200 digits).
