240. Search a 2D Matrix II

Matrix is sorted such that every row is ascending left → right and every column is ascending top → bottom. Return true if target appears anywhere in the matrix. (Unlike LC 74, rows are not one big sorted sequence — you can't just binary-search flatten.)

Microsoft · Medium

Summary

Hint: Start at a corner where one direction makes values go up and the other makes them go down — the top-right (or bottom-left) corner is the sweet spot. From the top-right: moving left shrinks the value, moving down grows it. So comparing matrix[row][col] to target gives you exactly one deterministic direction to move in. Each step eliminates one full row or one full column, so the whole search is O(m + n).

Answers

Naive — scan every cell

Ignore the sortedness completely and just brute-force every element.

Solution.java — brute force 
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;

        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (matrix[i][j] == target)
                    return true;
            }
        }
        return false;
    }
}
Works but throws away the whole reason the matrix was given to you sorted. O(m · n) time.

✅ Optimal — start at top-right, walk in a staircase

Put a finger at matrix[0][n - 1]. At each step compare to target:

  • if val == target: found ✓
  • if val > target: the whole current column below has values ≥ val > target, so this column can be thrown away — move left (col--).
  • if val < target: the whole current row to the left has values ≤ val < target, so this row can be thrown away — move down (row++).

We exit the matrix (either row == m or col < 0) once every row or column has been eliminated, so we return false.

✅ Solution.java — staircase from top-right 
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;

        int row = 0;
        int col = n - 1; // start from top-right

        while (row < m && col >= 0) {
            int val = matrix[row][col];

            if (val == target) {
                return true;
            } else if (val > target) {
                col--; // move left
            } else {
                row++; // move down
            }
        }

        return false;
    }
}
Why top-right (or bottom-left)? At the top-right, the current cell sits at the maximum of its row and the minimum of its column. So the comparison to target uniquely tells us whether to drop the column (it's too big) or drop the row (it's too small). If you started at the top-left or bottom-right, both neighbours in the two candidate directions move the same way relative to the current value, so the comparison wouldn't split the decision cleanly.

Walkthrough — search 5 in

1 4 7 11 15 2 5 8 12 19 3 6 9 16 22 10 13 14 17 24 18 21 23 26 30

Start at row=0, col=4 (value 15).

step | row col | val | compare with 5 | move | next (row, col) -----+---------+-----+----------------+------------+----------------- 0 | 0 4 | 15 | 15 > 5 | col-- | (0, 3) 1 | 0 3 | 11 | 11 > 5 | col-- | (0, 2) 2 | 0 2 | 7 | 7 > 5 | col-- | (0, 1) 3 | 0 1 | 4 | 4 < 5 | row++ | (1, 1) 4 | 1 1 | 5 | 5 == 5 | return true

Five comparisons, one hit. If instead we searched for 20: after step 3 we'd keep row-stepping down column 1 until we fall off the bottom, returning false — still bounded by m + n steps.

Complexity

Approach Time Space
Naive — brute-force scan O(m · n) O(1)
✅ Staircase from top-right O(m + n) — each iteration drops one row or one column O(1)

Because we only ever move left or down, we make at most (m - 1) + (n - 1) + 1 steps before exiting — hence the linear bound.
