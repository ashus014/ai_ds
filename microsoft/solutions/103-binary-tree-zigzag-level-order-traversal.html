103. Binary Tree Zigzag Level Order Traversal

Standard level-order BFS, but alternate directions per level: level 0 goes left → right, level 1 goes right → left, level 2 left → right, and so on. Return one inner list per level.

Microsoft · Medium

Summary

Hint: Run a normal BFS by levels (snapshot q.size() at the start of each level so you process exactly that level's nodes). Keep a boolean flag that flips each level. Either build the level left → right and reverse it on flipped levels, or — better — store the level in a deque and append to the correct end so no reversal is needed.

Answers

✅ BFS by levels + reverse on alternate levels

✅ Solution.java — BFS + reverse 
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
        List<List<Integer>> ans = new ArrayList<>();
        if (root == null) return ans;

        Queue<TreeNode> q = new LinkedList<>();
        q.offer(root);

        boolean reverse = false;

        while (!q.isEmpty()) {
            int size = q.size();
            List<Integer> level = new ArrayList<>();

            for (int i = 0; i < size; i++) {
                TreeNode node = q.poll();
                level.add(node.val);

                if (node.left != null) q.offer(node.left);
                if (node.right != null) q.offer(node.right);
            }

            if (reverse) {
                Collections.reverse(level); // extra work
            }

            ans.add(level);
            reverse = !reverse;
        }

        return ans;
    }
}
Why int size = q.size() at the top of each loop? That snapshots how many nodes belong to the current level, before we start enqueuing the next level's children. Without it, the same loop would bleed into the next level and the zigzag tracking would break.

Optional optimal — build the level in correct order, no reverse

💡 Idea. Instead of reversing later… build the level in the correct order while inserting.

🧠 Trick. Use a LinkedList<Integer> as a deque:

  • normal level → addLast()
  • reverse level → addFirst()

BFS still walks the tree left → right (so the queue stays the same); only the way we drop values into the level list flips per level. No Collections.reverse call, no extra O(k) pass per level.

Solution.java — BFS + deque insert 
class Solution {
    public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
        List<List<Integer>> ans = new ArrayList<>();
        if (root == null) return ans;

        Queue<TreeNode> q = new LinkedList<>();
        q.offer(root);

        boolean leftToRight = true;

        while (!q.isEmpty()) {
            int size = q.size();
            LinkedList<Integer> level = new LinkedList<>();

            for (int i = 0; i < size; i++) {
                TreeNode node = q.poll();

                if (leftToRight) {
                    level.addLast(node.val);
                } else {
                    level.addFirst(node.val);
                }

                if (node.left != null) q.offer(node.left);
                if (node.right != null) q.offer(node.right);
            }

            ans.add(level);
            leftToRight = !leftToRight;
        }

        return ans;
    }
}
Why a LinkedList? We need O(1) front insertion. ArrayList.add(0, x) would shift every existing element each time and degrade an entire reversed level to O(k²). LinkedList implements Deque, so addFirst and addLast are both O(1).

Complexity

Let n be the number of nodes in the tree.

Approach Time Space
✅ BFS + Collections.reverse on alternate levels O(n) — every node enqueued / dequeued once; reversals across all levels still sum to O(n) O(n) output + up to O(n) for the queue (widest level can be ~n/2)
BFS + LinkedList deque (addFirst / addLast) O(n) — each insert is O(1), no reversal pass O(n) output + up to O(n) queue (same as above)

Both are linear; the deque version just shaves the constant factor by skipping the per-level reverse pass.
