295. Find Median from Data Stream

Design a data structure supporting addNum(int) in roughly O(log n) and findMedian() in O(1). The canonical design splits the stream into two heaps — a max-heap for the lower half and a min-heap for the upper half — keeping them balanced within one element.

Microsoft · Hard — 14 April 2026

Summary

Hint: Two heaps — maxHeap holds the smaller half (median or smaller is on top), minHeap holds the larger half. Invariants: |maxHeap| = |minHeap| or |maxHeap| = |minHeap| + 1, and every value in maxHeap is ≤ every value in minHeap. The median is the top of maxHeap (odd) or the average of the two tops (even).

Answer

✅ Two heaps (max-heap + min-heap)

✅ MedianFinder.java — two heaps 
class MedianFinder {
    private PriorityQueue<Integer> maxHeap; // stores the smaller half (max-heap)
    private PriorityQueue<Integer> minHeap; // stores the larger half (min-heap)

    public MedianFinder() {
        maxHeap = new PriorityQueue<>((a, b) -> b - a); // max-heap
        minHeap = new PriorityQueue<>(); // min-heap
    }

    public void addNum(int num) {
        // Add to maxHeap first
        maxHeap.offer(num);

        // Move the largest from maxHeap to minHeap to maintain order
        minHeap.offer(maxHeap.poll());

        // Balance the heaps so maxHeap size is either equal to minHeap size
        // or one greater
        if (maxHeap.size() < minHeap.size()) {
            maxHeap.offer(minHeap.poll());
        }
    }

    public double findMedian() {
        if (maxHeap.size() > minHeap.size()) {
            return maxHeap.peek();
        } else {
            return (maxHeap.peek() + minHeap.peek()) / 2.0;
        }
    }
}

/**
 * Your MedianFinder object will be instantiated and called as such:
 * MedianFinder obj = new MedianFinder();
 * obj.addNum(num);
 * double param_2 = obj.findMedian();
 */
Why the push-then-rebalance pattern? Pushing into maxHeap first and then shifting its top into minHeap guarantees the cross-heap ordering without comparing against peeks (which would require null checks for an empty minHeap). The final if restores the size invariant. A more direct variant uses explicit comparisons on num vs maxHeap.peek() but needs extra edge-case handling for the first insert.

Dry run — stream 1, 2, 3

  • addNum(1): push 1 to max, move 1 to min. |max| < |min| ⇒ move 1 back to max. State: max=[1], min=[]. Median = 1.
  • addNum(2): push 2 to max → max=[2,1], move 2 to min → max=[1], min=[2]. Sizes equal, no rebalance. Median = (1 + 2) / 2 = 1.5.
  • addNum(3): push 3 to max → max=[3,1], move 3 to min → max=[1], min=[2,3]. |max| < |min| ⇒ move 2 back to max. State: max=[2,1], min=[3]. Median = 2.

Complexity

  • addNum: O(log n) — each call does a constant number of heap pushes/pops.
  • findMedian: O(1) — two peeks.
  • Space: O(n) — the two heaps collectively store all inserted numbers.
  • Follow-ups. If all numbers lie in [0, 100], a 101-bucket count array + running totals gives O(1) amortised median. For 99% of values in [0, 100] and the rest outside, hybridise: count array for the common band + a balanced BST/order-statistics tree for outliers.
