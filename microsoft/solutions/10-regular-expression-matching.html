10. Regular Expression Matching

Match an input string s against a pattern p with . (any single char) and * (zero or more of the preceding element). Try “skip x*” vs “consume one char and repeat” when the next pattern char is *. Problem on LeetCode →

Microsoft · Hard — 14 April 2026

Summary

Hint: If the next pattern character is *, either skip the x* pair (notTake) or, if the current chars match, consume one character from s and stay on the same pattern index (take). Otherwise require a single-character match and advance both pointers.

Answers

✅ Recursive — substring

✅ Solution.java — substring 
class Solution {

    public boolean solve(String s, String p) {
        // 🔹 Base case:
        // If pattern is empty, string must also be empty
        if (p.length() == 0) {
            return s.length() == 0;
        }

        // 🔹 Check if first character matches
        boolean firstCharMatched = false;

        if (s.length() > 0 &&
           (p.charAt(0) == s.charAt(0) || p.charAt(0) == '.')) {
            firstCharMatched = true;
        }

        // 🔹 Case when next character in pattern is '*'
        if (p.length() >= 2 && p.charAt(1) == '*') {

            // ❌ NOT TAKE:
            // Skip "x*" → move pattern by 2
            boolean notTake = solve(s, p.substring(2));

            // ✅ TAKE:
            // If current char matches, consume 1 char from string
            // and stay on same pattern (because '*' can repeat)
            boolean take = firstCharMatched && solve(s.substring(1), p);

            return notTake || take;
        }

        // 🔹 Normal case (no '*'):
        // If current char matches → move both forward
        return firstCharMatched &&
               solve(s.substring(1), p.substring(1));
    }

    public boolean isMatch(String s, String p) {
        return solve(s, p);
    }
}

Recursive — indices (no substring)

Solution.java — indices 
class Solution {

    public boolean solve(int i, int j, String s, String p) {

        // 🔹 If pattern finished → string must also finish
        if (j == p.length()) {
            return i == s.length();
        }

        // 🔹 Check if current chars match
        boolean firstCharMatched = false;

        if (i < s.length() &&
           (s.charAt(i) == p.charAt(j) || p.charAt(j) == '.')) {
            firstCharMatched = true;
        }

        // 🔹 If next char is '*'
        if (j + 1 < p.length() && p.charAt(j + 1) == '*') {

            // ❌ NOT TAKE → skip "x*"
            boolean notTake = solve(i, j + 2, s, p);

            // ✅ TAKE → consume one char from s, stay on same pattern
            boolean take = firstCharMatched && solve(i + 1, j, s, p);

            return notTake || take;
        }

        // 🔹 Normal match → move both pointers
        return firstCharMatched && solve(i + 1, j + 1, s, p);
    }

    public boolean isMatch(String s, String p) {
        return solve(0, 0, s, p);
    }
}

Memoised (same recurrence)

Solution.java — memo 
class Solution {

    Boolean[][] dp;

    boolean solve(int i, int j, String s, String p) {

        // 🔹 check cache
        if (dp[i][j] != null) {
            return dp[i][j];
        }

        // 🔹 base case
        if (j == p.length()) {
            return dp[i][j] = (i == s.length());
        }

        boolean first_char_matched = false;

        if (i < s.length() &&
           (p.charAt(j) == s.charAt(i) || p.charAt(j) == '.')) {
            first_char_matched = true;
        }

        boolean ans;

        // 🔹 '*' case
        if (j + 1 < p.length() && p.charAt(j + 1) == '*') {

            boolean not_take = solve(i, j + 2, s, p);
            boolean take = first_char_matched && solve(i + 1, j, s, p);

            ans = not_take || take;
        } else {
            ans = first_char_matched && solve(i + 1, j + 1, s, p);
        }

        // 🔹 store result
        return dp[i][j] = ans;
    }

    public boolean isMatch(String s, String p) {

        // size +1 because i and j can reach end
        dp = new Boolean[s.length() + 1][p.length() + 1];

        return solve(0, 0, s, p);
    }
}
Substring DFS is simplest to read but allocates new strings. Index + memo is what you want for interviews: O(m n) states, O(m n) time.

Complexity

  • Substring recursion: exponential time in the worst case without memo; extra space from substring copies.
  • Memoised (i, j): time O(m n), space O(m n) for |s| = m, |p| = n.
