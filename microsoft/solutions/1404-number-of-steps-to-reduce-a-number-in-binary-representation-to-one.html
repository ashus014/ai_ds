1404. Number of Steps to Reduce a Number in Binary Representation to One

Given a positive binary string s, count how many steps it takes to reduce it to "1" under the rules: if the number is even, divide by 2; if odd, add 1. (In binary: even means last bit is 0, odd means last bit is 1.)

Microsoft · Medium

Summary

Hint: Work in binary directly, right → left. "Divide by 2" is just dropping the last bit; "add 1" creates a carry that ripples up. For each bit (except the leading 1), combine it with an incoming carry:

  • bit + carry == 0 (even here) ⇒ 1 step to divide, carry stays 0.
  • bit + carry == 1 (odd here) ⇒ 2 steps (add 1, then divide), carry becomes 1.
  • bit + carry == 2 (even here, with a propagating carry) ⇒ 1 step to divide, carry stays 1.

At the end, add carry itself (if the leading 1 absorbs a carry, that's one extra "divide" to collapse 10 → 1).

Answers

Naive — simulate the string

Keep the binary as a StringBuilder. On each step: if last char is 0, pop it (divide by 2). Otherwise ripple-add 1 by flipping trailing 1s to 0s and setting the first 0 (or prepending a new 1 if we blew past the front).

Solution.java — simulate the string 
class Solution {
    public int numSteps(String s) {
        StringBuilder sb = new StringBuilder(s);
        int steps = 0;

        while (!sb.toString().equals("1")) {
            if (sb.charAt(sb.length() - 1) == '0') {
                // even → divide by 2
                sb.deleteCharAt(sb.length() - 1);
            } else {
                // odd → add 1
                int i = sb.length() - 1;

                while (i >= 0 && sb.charAt(i) == '1') {
                    sb.setCharAt(i, '0');
                    i--;
                }

                if (i >= 0) {
                    sb.setCharAt(i, '1');
                } else {
                    sb.insert(0, '1');
                }
            }
            steps++;
        }

        return steps;
    }
}
Correct but expensive — each "add 1" can ripple up the whole string, giving worst-case O(n²) with extra string work.

✅ Optimal — right-to-left single pass with a running carry

Walk bits from index n - 1 down to 1 (we deliberately stop before the leading bit s[0], since that's the 1 we're trying to end up with). At every position, look at bit + carry:

  • 0: current position is 0 with no incoming carry — one shift-right step.
  • 1: current position behaves odd — we add 1 (that flips this to 0 and sends a carry up), then shift right. That's 2 steps and a new carry = 1.
  • 2: current bit is 1 and incoming carry is 1 — they sum to 10, so this position is effectively 0 with a carry to the next. One shift-right; carry stays 1.

After the loop, the leading bit is 1. If a carry is still floating in, we have 1 + 1 = 10, which needs one more divide to become 1. Hence return steps + carry.

✅ Solution.java — single pass + carry 
class Solution {
    public int numSteps(String s) {
        int steps = 0;
        int carry = 0;

        // traverse from right to left (ignore first bit)
        for (int i = s.length() - 1; i > 0; i--) {
            int bit = s.charAt(i) - '0';

            if (bit + carry == 1) {
                // odd
                steps += 2;
                carry = 1;
            } else {
                // even
                steps += 1;
            }
        }

        // if carry remains, one extra step
        return steps + carry;
    }
}
Why the else branch doesn't reset carry. When bit + carry is 0 the carry was already 0, so "stay at 0" is correct. When it's 2 we intentionally want carry to stay 1 — the two 1s added to 10 in binary, so a carry propagates upward. Leaving the variable alone captures both cases.

Walkthrough — s = "1101" (decimal 13)

Sanity check by simulation first:

13 odd → +1 → 14 step 1 14 even → /2 → 7 step 2 7 odd → +1 → 8 step 3 8 even → /2 → 4 step 4 4 even → /2 → 2 step 5 2 even → /2 → 1 step 6 ✓

Now the optimal pass (i goes from 3 down to 1, never 0):

i | s[i] | carry_in | bit+carry | branch | steps | carry_out --+------+----------+-----------+----------+-------+---------- 3 | 1 | 0 | 1 | odd (+2) | 2 | 1 2 | 0 | 1 | 1 | odd (+2) | 4 | 1 1 | 1 | 1 | 2 | even(+1) | 5 | 1 (carry stays 1) loop ends. return steps + carry = 5 + 1 = 6 ✓
