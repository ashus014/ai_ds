1615. Maximal Network Rank

You have n cities and undirected roads between pairs. The network rank of two different cities is how many distinct roads are incident to at least one of them (a single road between the two counts once). Compute the maximum rank over all unordered pairs.

Microsoft · Medium

Summary

Hint: Let degree[i] be the number of roads touching city i. For any pair i < j, if there is no road between them, all roads at i and j are disjoint, so rank = degree[i] + degree[j]. If they are directly connected, that one road was counted twice, so subtract 1. Enumerate all pairs and take the maximum.

Answer

✅ Degrees + adjacency check per pair

✅ Solution.java — degrees + pairs 
import java.util.*;

class Solution {
    public int maximalNetworkRank(int n, int[][] roads) {

        int[] degree = new int[n];

        // adjacency list using List<List<Integer>>
        List<List<Integer>> adj = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            adj.add(new ArrayList<>());
        }

        // build graph
        for (int[] road : roads) {
            int u = road[0];
            int v = road[1];

            degree[u]++;
            degree[v]++;

            adj.get(u).add(v);
            adj.get(v).add(u);
        }

        int maxRank = 0;

        // check all pairs
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {

                int rank = degree[i] + degree[j];

                // check if directly connected
                if (adj.get(i).contains(j)) {
                    rank--;
                }

                maxRank = Math.max(maxRank, rank);
            }
        }

        return maxRank;
    }
}
Double-counting the shared edge. Summing degrees counts an i–j road once from each endpoint. The contains check detects that case so you subtract one. With larger n, storing neighbors in a HashSet per city makes the edge test O(1) instead of scanning a list.

Complexity

  • Time: O(n + m) to build degrees and adjacency lists (m = roads.length), plus O(n²) pairs; each contains is O(degree(i)), so worst case about O(n³) when the graph is dense — fine for LeetCode constraints (n ≤ 100).
  • Space: O(n + m) for the degree array and adjacency lists.
