285. Inorder Successor in BST

Given a BST and a node x, return the value that comes immediately after x.data in an inorder traversal — or -1 if x is the largest. Naive route: dump the inorder into a list and look up x's index; optimal route exploits the BST property for O(h) time.

Microsoft · Medium

Summary

Hint (naive): Do a regular inorder traversal and dump the values into a list. Scan the list to find x.data and return the next entry (or -1).

Optimal (BST property): Walk the tree from root once, remembering the last ancestor where you turned left (i.e., the last node strictly greater than x.data). That ancestor is the inorder successor. O(h) time, O(1) extra space.

Answer

Naive approach — inorder dump + linear scan

Solution.java — inorder dump 
/*Complete the function below
Node is as follows:
class Node{
    int data;
    Node left,right;
    Node(int d){
        data=d;
        left=right=null;
    }
}
*/

class Solution {

    public int inorderSuccessor(Node root, Node x) {
        List<Integer> list = new ArrayList<>();

        // Step 1: store inorder
        inorder(root, list);

        // Step 2: find x and return next
        for (int i = 0; i < list.size(); i++) {
            if (list.get(i) == x.data) {
                // if next exists
                if (i + 1 < list.size()) {
                    return list.get(i + 1);
                } else {
                    return -1;
                }
            }
        }

        return -1;
    }

    // simple inorder traversal
    private void inorder(Node root, List<Integer> list) {
        if (root == null) return;

        inorder(root.left, list);
        list.add(root.data);
        inorder(root.right, list);
    }
}
Works in O(n) time / O(n) extra space, and it ignores the BST structure entirely — it'd work on any binary tree. The two BST-aware solutions below cut this to O(h) time with O(1) / O(h) extra space.

✅ Optimal — iterative descent (last-left-turn ancestor)

✅ Solution.java — iterative 
class Solution {
    public int inorderSuccessor(Node root, Node x) {
        Node successor = null;

        while (root != null) {
            if (x.data < root.data) {
                successor = root;   // possible answer
                root = root.left;  // try smaller one
            } else {
                root = root.right; // move right
            }
        }

        return (successor != null) ? successor.data : -1;
    }
}
Why it works. Starting from the root, every time we go left, the current node is a candidate (it's strictly greater than x.data and we may yet find a smaller one). Every time we go right, all of root's ancestors via right turns are not greater than x.data, so we discard them. The last node where we went left is the smallest value strictly greater than x.data — exactly the inorder successor.

✅ Optimal — recursive descent (same idea)

✅ Solution.java — recursive 
class Solution {

    Node ans = null;

    public int inorderSuccessor(Node root, Node x) {
        helper(root, x);
        return (ans != null) ? ans.data : -1;
    }

    private void helper(Node root, Node x) {
        if (root == null) return;

        if (root.data > x.data) {
            ans = root;          // store candidate
            helper(root.left, x); // try to find smaller one
        } else {
            helper(root.right, x); // go right
        }
    }
}
Same logic as the iterative version, written as a single descent. The instance field ans records the latest "go-left" node; we never need to backtrack.

Complexity

  • Naive (inorder dump): Time O(n), Space O(n) — visits every node and stores the full list.
  • Optimal (iterative): Time O(h), Space O(1) — single descent from root to a leaf.
  • Optimal (recursive): Time O(h), Space O(h) — single descent on the call stack, balanced BST: O(log n); skewed: O(n).
