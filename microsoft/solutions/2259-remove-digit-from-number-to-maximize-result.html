2259. Remove Digit From Number to Maximize Result

Given a numeric string number and a character digit appearing in it, remove exactly one occurrence of digit so the resulting number is as large as possible. The naive way tries every removal and keeps the lexicographically largest; the optimal way runs in a single pass with one greedy rule.

Microsoft · Easy — 14 April 2026

Summary

Hint: Since all candidate strings have the same length, lexicographic comparison equals numeric comparison. Greedy rule: remove the first occurrence of digit that is followed by a strictly larger digit (then a bigger digit shifts left ⇒ bigger number). If no such position exists, removing the last occurrence is best — every position has only ≤ digits to its right, so we want to lose a low digit as far right as possible.

Answers

Naive approach — try every removal, keep the max

Solution.java — try every removal 
class Solution {
    public String removeDigit(String number, char digit) {
        String best = "";

        for (int i = 0; i < number.length(); i++) {
            if (number.charAt(i) == digit) {
                String candidate = number.substring(0, i) + number.substring(i + 1);

                if (best.equals("") || candidate.compareTo(best) > 0) {
                    best = candidate;
                }
            }
        }

        return best;
    }
}
Cost. Up to O(n) candidate strings, each O(n) to build and compare ⇒ O(n²) time / O(n) per candidate space. Trivially correct, fine for n ≤ 100, but the greedy below is O(n).

✅ Optimal — greedy single pass (last occurrence as fallback)

✅ Solution.java — greedy single pass 
class Solution {
    public String removeDigit(String number, char digit) {
        int index = -1;

        for (int i = 0; i < number.length(); i++) {
            if (number.charAt(i) == digit) {
                index = i; // store last occurrence

                // greedy condition
                if (i + 1 < number.length() && number.charAt(i) < number.charAt(i + 1)) {
                    return number.substring(0, i) + number.substring(i + 1);
                }
            }
        }

        // if no greedy case found → remove last occurrence
        return number.substring(0, index) + number.substring(index + 1);
    }
}
Why greedy works. All candidates have the same length, so the most-significant digit that changes wins. Removing position i shifts number[i+1] into slot i. If number[i+1] > digit, doing the removal earlier maximises that high-order slot — pick the first such i. If no occurrence is followed by a strictly larger digit, every removal makes the high-order positions weakly worse; the least damage is to remove the rightmost match (any earlier removal would shift an equal-or-smaller digit into a more significant slot).

Dry run — number = "1231", digit = '1'

  • i=0: '1' < '2' ⇒ greedy fires, return "231". ✅

Dry run — number = "551", digit = '5'

  • i=0: '5' followed by '5', not strictly greater. index=0.
  • i=1: '5' followed by '1', not greater. index=1.
  • Loop ends → remove last occurrence at 1 ⇒ "51". ✅

Complexity

  • Naive: Time O(n²), Space O(n) per candidate string.
  • Optimal: Time O(n), Space O(n) for the returned string only — single pass and a constant amount of extra state.
