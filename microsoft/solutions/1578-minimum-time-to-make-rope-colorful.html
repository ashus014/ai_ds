1578. Minimum Time to Make Rope Colorful

Given a rope of balloons colors and costs neededTime[i] to remove balloon i, remove the fewest-cost balloons so no two adjacent balloons share a color. Inside any run of equal colors, keep the most expensive one and remove the rest — a classic "sum − max of each group" trick.

Microsoft · Medium — 14 April 2026

Summary

Hint: Group contiguous equal colors. Within each group we must remove all but one balloon; to minimise cost, keep the one with the highest neededTime (add sum − max to the answer). This can be done in a one-pass sweep keeping only the running max of the current group.

Answers

Naive approach — group scan (sum − max per run)

Solution.java — group scan 
class Solution {
    public int minCost(String colors, int[] neededTime) {
        int n = colors.length();
        int totalCost = 0;

        int i = 0;

        while (i < n) {
            int j = i;
            int sum = 0;
            int max = 0;

            // group of same colors
            while (j < n && colors.charAt(i) == colors.charAt(j)) {
                sum += neededTime[j];
                max = Math.max(max, neededTime[j]);
                j++;
            }

            totalCost += (sum - max);
            i = j;
        }

        return totalCost;
    }
}
Clear and obviously correct — explicitly computes each group's sum and max, then adds sum − max (the total cost of removing everything except the most expensive). Same asymptotic cost as the optimal version; just uses more memory semantics per group.

✅ Optimal — one-pass, rolling max of the current run

✅ Solution.java — rolling max 
class Solution {
    public int minCost(String colors, int[] neededTime) {
        int totalCost = 0;
        int prevMax = neededTime[0];

        for (int i = 1; i < colors.length(); i++) {
            if (colors.charAt(i) == colors.charAt(i - 1)) {
                totalCost += Math.min(prevMax, neededTime[i]);
                prevMax = Math.max(prevMax, neededTime[i]);
            } else {
                prevMax = neededTime[i];
            }
        }

        return totalCost;
    }
}
Why it works. Inside a run, every time we see a duplicate we must delete either the rolling max or the new balloon. The cheaper of the two goes (Math.min); the survivor becomes the new max (Math.max). When the color changes, we reset prevMax and start a new run. This is algebraically equivalent to sum − max per group, computed on the fly.

Dry run — colors = "abaac", neededTime = [1,2,3,4,5]

  • i=1 'b' vs 'a': reset prevMax = 2.
  • i=2 'a' vs 'b': reset prevMax = 3.
  • i=3 'a' vs 'a': duplicate — add min(3, 4) = 3, prevMax = 4. Cost = 3.
  • i=4 'c' vs 'a': reset prevMax = 5. Final cost = 3 ✅.

Complexity

  • Time: O(n) — single pass over the balloons.
  • Space: O(1) — two integer accumulators (totalCost, prevMax).
