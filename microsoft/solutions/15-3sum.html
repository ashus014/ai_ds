15. 3Sum

Find all unique triplets in the array that sum to zero. Start from brute force, then improve with hashing, and finally use sort + two pointers for the optimal approach.

Microsoft · Medium — 16 April 2026

Summary

Hint: For optimal performance, sort the array, fix one index, and solve the rest with two pointers while skipping duplicates.

Answers

Naive approach

Solution.java — naive 
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> set = new HashSet<>();
        int n = nums.length;

        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                for (int k = j + 1; k < n; k++) {

                    if (nums[i] + nums[j] + nums[k] == 0) {
                        List<Integer> temp = Arrays.asList(nums[i], nums[j], nums[k]);
                        Collections.sort(temp); // to avoid duplicate order
                        set.add(temp);
                    }
                }
            }
        }

        return new ArrayList<>(set);
    }
}
Brute force triples. Time O(n^3), space depends on unique triplets stored.

Better approach (HashSet)

Solution.java — hashset 
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> result = new HashSet<>();
        int n = nums.length;

        for (int i = 0; i < n; i++) {
            Set<Integer> seen = new HashSet<>();

            for (int j = i + 1; j < n; j++) {
                int target = -(nums[i] + nums[j]);

                if (seen.contains(target)) {
                    List<Integer> temp = Arrays.asList(nums[i], nums[j], target);
                    Collections.sort(temp); // handle duplicates
                    result.add(temp);
                }

                seen.add(nums[j]);
            }
        }

        return new ArrayList<>(result);
    }
}
Pair + complement per fixed i. Average time around O(n^2) with hashing, plus dedupe cost.

✅ Optimal approach

✅ Solution.java — optimal 
class Solution {

    List<List<Integer>> result = new ArrayList<>();

    public List<List<Integer>> threeSum(int[] nums) {

        int n = nums.length;
        if (n < 3) return result;

        Arrays.sort(nums);

        // fixing one element n1
        for (int i = 0; i < n - 2; i++) {

            // skip duplicate n1
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int n1 = nums[i];
            int target = -n1;

            // find n2 and n3 using twoSum
            twoSum(nums, target, i + 1, n - 1, n1);
        }

        return result;
    }

    private void twoSum(int[] nums, int target, int left, int right, int n1) {

        while (left < right) {

            int sum = nums[left] + nums[right];

            if (sum == target) {

                result.add(Arrays.asList(n1, nums[left], nums[right]));

                left++;
                right--;

                // skip duplicate n2
                while (left < right && nums[left] == nums[left - 1]) {
                    left++;
                }

                // skip duplicate n3
                while (left < right && nums[right] == nums[right + 1]) {
                    right--;
                }

            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
    }
}
Sort + two pointers with duplicate skipping. Time O(n^2) after sorting, space O(1) extra (excluding output).

Complexity

  • Naive: Time O(n^3), extra space for set of answers.
  • HashSet better: Average Time near O(n^2), extra set space per fixed index plus output dedupe storage.
  • Optimal (sort + two pointers): Time O(n^2) overall, auxiliary space O(1) (excluding output).
