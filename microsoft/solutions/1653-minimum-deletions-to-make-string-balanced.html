1653. Minimum Deletions to Make String Balanced

Given a string of 'a' and 'b', return the minimum deletions so that no 'b' appears before any 'a'. Equivalently, the surviving string must look like aaaa…bbbb: some prefix of 'a's followed by some suffix of 'b's.

Microsoft · Medium

Summary

Hint: Imagine cutting the string at index i — keep 'a's on the left and 'b's on the right. Cost of that cut is (b's in left) + (a's in right). The naive solution tries every cut. The optimal one collapses both halves into a single left-to-right scan with two counters: how many 'b' we have already seen, and the running deletion budget.

Answers

Naive — try every split point

Solution.java — try every split 
class Solution {
    public int minimumDeletions(String s) {
        int n = s.length();
        int ans = Integer.MAX_VALUE;

        for (int i = 0; i <= n; i++) { // split point
            int delete = 0;

            // left side: count 'b'
            for (int j = 0; j < i; j++) {
                if (s.charAt(j) == 'b') delete++;
            }

            // right side: count 'a'
            for (int j = i; j < n; j++) {
                if (s.charAt(j) == 'a') delete++;
            }

            ans = Math.min(ans, delete);
        }

        return ans;
    }
}
Why this is O(n²). For every split point i (there are n + 1 of them), we re-scan both halves. The optimal version below removes the inner loop by maintaining the two counts incrementally.

✅ Optimal — single pass with two counters

✅ Solution.java — single pass 
class Solution {
    public int minimumDeletions(String s) {
        int bCount = 0;
        int deletions = 0;

        for (char ch : s.toCharArray()) {
            if (ch == 'b') {
                bCount++;
            } else {
                deletions = Math.min(deletions + 1, bCount);
            }
        }

        return deletions;
    }
}

Why the single-pass works

We scan the string left → right and maintain just two things:

  • bCount → how many 'b' we have seen so far.
  • deletions → minimum deletions needed for the prefix processed up to now.

What happens at each character

If it is 'b' — no problem. bCount++. A 'b' on the left side is fine for now (it can sit in the final bbbb tail).

If it is 'a' — this is the tricky case. Every previous 'b' sits to its left, which breaks balance, so we have two choices:

  • 👉 Delete this 'a' → cost becomes deletions + 1.
  • 👉 Or delete every previous 'b' → cost becomes bCount.

We always take the cheaper option, so:

deletions = Math.min(deletions + 1, bCount);

Why this stays optimal as the prefix grows. The state we keep, (bCount, deletions), is exactly enough to extend either decision. If a future 'a' arrives, both branches above can be evaluated again from the same numbers — there is no benefit to remembering which 'a's or 'b's we deleted, only the totals.

Complexity

  • Time (optimal): O(n) — one pass over the string.
  • Time (naive): O(n²) — split point × two-half scan.
  • Space: O(1) in both, just two integer counters.
