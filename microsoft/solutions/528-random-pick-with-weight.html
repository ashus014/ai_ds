528. Random Pick with Weight

Given an array w where w[i] is the weight of index i, implement pickIndex() so it returns index i with probability w[i] / sum(w).

Microsoft · Medium

Summary

Hint: Build a prefix sum of weights so prefix[i] = w[0]+…+w[i]. Pick a random target in [1, total] and return the smallest index i with prefix[i] ≥ target. Linear scan works; binary search makes it O(log n) per call.

Answers

Naive approach — expanded bucket

Solution.java — bucket 
class Solution {

    List<Integer> bucket;
    Random rand;

    public Solution(int[] w) {
        bucket = new ArrayList<>();
        rand = new Random();

        for (int i = 0; i < w.length; i++) {
            for (int j = 0; j < w[i]; j++) {
                bucket.add(i);
            }
        }
    }

    public int pickIndex() {
        int idx = rand.nextInt(bucket.size()); // 0 to totalWeight-1
        return bucket.get(idx);
    }
}

/**
 * Your Solution object will be instantiated and called as such:
 * Solution obj = new Solution(w);
 * int param_1 = obj.pickIndex();
 */
Heads-up. Each weight w[i] can be up to 1e5 and the sum up to 1e8, so the bucket can blow past memory limits. Keep this as intuition only.

Optimal — prefix sum + linear scan

Solution.java — prefix sum (linear) 
import java.util.*;

class Solution {

    int[] prefix;
    int total;
    Random rand;

    public Solution(int[] w) {
        prefix = new int[w.length];
        rand = new Random();

        prefix[0] = w[0];
        for (int i = 1; i < w.length; i++) {
            prefix[i] = prefix[i - 1] + w[i];
        }

        total = prefix[w.length - 1];
    }

    public int pickIndex() {
        int target = rand.nextInt(total) + 1;

        for (int i = 0; i < prefix.length; i++) {
            if (prefix[i] >= target) {   // ✅ found range
                return i;
            }
        }

        return -1; // won't happen
    }
}
Right idea, wrong complexity. Storing prefix sums is perfect, but scanning linearly on every pickIndex() becomes O(n) per call. Good stepping stone before the binary-search version.

✅ Optimal — prefix sum + binary search

✅ Solution.java — prefix sum + binary search 
import java.util.*;

class Solution {

    int[] prefix;
    int total;
    Random rand;

    public Solution(int[] w) {
        prefix = new int[w.length];
        rand = new Random();

        prefix[0] = w[0];
        for (int i = 1; i < w.length; i++) {
            prefix[i] = prefix[i - 1] + w[i];
        }

        total = prefix[w.length - 1];
    }

    public int pickIndex() {
        int target = rand.nextInt(total) + 1;

        int left = 0, right = prefix.length - 1;

        while (left < right) {
            int mid = left + (right - left) / 2;

            if (prefix[mid] >= target) {
                right = mid;      // ✅ go left
            } else {
                left = mid + 1;   // ✅ go right
            }
        }

        return left; // ✅ answer
    }
}
Classic lower-bound binary search. We want the smallest i with prefix[i] ≥ target, so on a match we move right = mid (keep the candidate) and otherwise left = mid + 1.

Complexity

  • Naive bucket: build O(sum(w)) time/space, pick O(1).
  • Prefix + linear scan: build O(n), pick O(n).
  • ✅ Prefix + binary search: build O(n), pick O(log n), space O(n).
