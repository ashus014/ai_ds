173. Binary Search Tree Iterator

Implement an iterator over a BST that returns the next smallest element in order. hasNext() and next() should run in average O(1) time and use O(h) memory, where h is the tree height — the classic pattern is a stack that simulates inorder “on demand.” Flattening the whole inorder into a list is simpler but uses O(n) extra space.

Microsoft · Medium

Summary

Hint (eager list): Run a full inorder traversal in the constructor, push every value into an ArrayList, then next() / hasNext() are just index bookkeeping — O(n) space upfront.

Hint (optimal stack): Keep a stack of nodes whose left spine has been pushed. next() pops the smallest not-yet-returned node, then pushes the left spine of its right child (if any). The stack never holds more than O(h) nodes.

Answers

Eager approach — full inorder in ArrayList (higher space)

Not “wrong,” just trades memory for simplicity: constructor does all the work; iteration is trivial. Fine when the tree is small or you know you will walk every node anyway.

BSTIterator.java — eager ArrayList 
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
import java.util.*;

class BSTIterator {

    List<Integer> list = new ArrayList<>();
    int index = 0;

    public BSTIterator(TreeNode root) {
        inorder(root);
    }

    public int next() {
        return list.get(index++); // return and move pointer
    }

    public boolean hasNext() {
        return index < list.size();
    }

    private void inorder(TreeNode node) {
        if (node == null) return;

        inorder(node.left);
        list.add(node.val);
        inorder(node.right);
    }
}

/**
 * Your BSTIterator object will be instantiated and called as such:
 * BSTIterator obj = new BSTIterator(root);
 * int param_1 = obj.next();
 * boolean param_2 = obj.hasNext();
 */
Space trade-off. You store every node value — O(n) integers plus O(h) recursion depth during the initial traversal. next() / hasNext() become O(1) after that, but the problem’s follow-up usually asks for O(h) auxiliary memory instead.

✅ Optimal approach — controlled stack (lazy inorder)

✅ BSTIterator.java — stack + pushLeft 
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
import java.util.*;

class BSTIterator {

    Stack<TreeNode> stack = new Stack<>();

    public BSTIterator(TreeNode root) {
        // push all left nodes
        pushLeft(root);
    }

    public int next() {
        // get next smallest element
        TreeNode node = stack.pop();

        // if right exists, process its left chain
        if (node.right != null) {
            pushLeft(node.right);
        }

        return node.val;
    }

    public boolean hasNext() {
        return !stack.isEmpty();
    }

    private void pushLeft(TreeNode node) {
        while (node != null) {
            stack.push(node);
            node = node.left;
        }
    }
}

/**
 * Your BSTIterator object will be instantiated and called as such:
 * BSTIterator obj = new BSTIterator(root);
 * int param_1 = obj.next();
 * boolean param_2 = obj.hasNext();
 */
Why it matches inorder. The stack always holds the “current left spine” from some subtree root down — the top is the next unvisited smallest. After you pop a node, its inorder successor is either the leftmost node of its right subtree (pushed by pushLeft) or whatever was already below it on the stack (an ancestor you deferred when you first went left).

Complexity

  • Eager ArrayList: Constructor O(n) time and O(n) space. Each next() / hasNext() is O(1) after that.
  • ✅ Stack (lazy): Constructor O(h) time to push the left spine from the root. Each next() is amortized O(1) over the whole traversal (each edge crossed at most twice); worst single call O(h). Space O(h) for the stack.

Comparison table

Approach Constructor next() / hasNext() Extra space
Eager — inorder → ArrayList O(n) O(1) O(n)
✅ Stack — lazy left spines O(h) amortized O(1) (worst O(h) per call) O(h)

n = number of nodes, h = height of the BST (balanced: h ≈ log n; skewed: h ≈ n).
