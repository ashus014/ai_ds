380. Insert Delete GetRandom O(1)

Design a set of distinct integers supporting insert(val), remove(val), and getRandom() — each in average O(1). getRandom must return any current element with uniform probability.

Microsoft · Medium

Summary

Hint: HashSet alone gives O(1) insert / remove but getRandom over a hash table can't pick uniformly in O(1). ArrayList alone gives O(1) random pick but remove(val) would be linear. Combine them:

  • An ArrayList<Integer> — the actual elements, packed contiguously. getRandom just picks a random index here.
  • A HashMap<Integer, Integer> mapping value → its index in the list. Lets us locate any element in O(1).

The trick to O(1) remove: swap the doomed element with the last element of the list, update the map for the moved element, then pop the last position. No shifting.

Answer

✅ ArrayList + HashMap + swap-with-last on remove

✅ RandomizedSet.java — list + map + swap-with-last 
import java.util.*;

class RandomizedSet {

    private List<Integer> list;
    private Map<Integer, Integer> map;
    private Random rand;

    public RandomizedSet() {
        list = new ArrayList<>();        // stores values
        map = new HashMap<>();          // value -> index
        rand = new Random();
    }

    public boolean insert(int val) {
        // if already present → return false
        if (map.containsKey(val)) {
            return false;
        }

        // add at end
        list.add(val);
        map.put(val, list.size() - 1);

        return true;
    }

    public boolean remove(int val) {
        // if not present → return false
        if (!map.containsKey(val)) {
            return false;
        }

        // get index of element to remove
        int index = map.get(val);

        // get last element
        int lastElement = list.get(list.size() - 1);

        // swap with last element
        list.set(index, lastElement);
        map.put(lastElement, index);

        // remove last element
        list.remove(list.size() - 1);

        // remove from map
        map.remove(val);

        return true;
    }

    public int getRandom() {
        // pick random index
        int randomIndex = rand.nextInt(list.size());
        return list.get(randomIndex);
    }
}

/**
 * Your RandomizedSet object will be instantiated and called as such:
 * RandomizedSet obj = new RandomizedSet();
 * boolean param_1 = obj.insert(val);
 * boolean param_2 = obj.remove(val);
 * int param_3 = obj.getRandom();
 */
Why the swap-with-last is the whole game. Removing from the middle of an ArrayList normally takes O(n) because everything to the right shifts left. By overwriting the doomed slot with whatever lives at the tail (and updating that element's entry in the map to its new index), the only thing left to do is pop the tail — which is O(1) for an ArrayList. The map keeps every value's location accurate after each operation, so all three ops stay O(1).

Walkthrough

Operations: insert(1), insert(2), insert(3), remove(2), insert(4), getRandom().

op | list (after) | map (after) --------------+----------------+-------------------- insert(1) | [1] | {1→0} insert(2) | [1, 2] | {1→0, 2→1} insert(3) | [1, 2, 3] | {1→0, 2→1, 3→2} remove(2) | [1, 3] | {1→0, 3→1} step a: index = map.get(2) = 1 step b: last = list.get(2) = 3 step c: list.set(1, 3) → [1, 3, 3] map.put(3, 1) → {1→0, 2→1, 3→1} step d: list.remove(2) → [1, 3] map.remove(2) → {1→0, 3→1} insert(4) | [1, 3, 4] | {1→0, 3→1, 4→2} getRandom() | returns one of {1, 3, 4} uniformly

Notice in remove(2) how the last element (3) jumps into position 1 and the map immediately updates 3 → 1. After that, list.remove(list.size() - 1) is a cheap pop.

Edge cases worth knowing

  • Removing the last element itself. The "swap" becomes a no-op (we set list.get(index) to itself and map.put(val, index) right before removing it from the map). Still correct, no special-case branch needed.
  • Duplicate insert. The map.containsKey guard returns false without touching the list; the list never holds duplicates so the random pick stays uniform.
  • Removing a missing value. Same guard returns false upfront — no list mutation.
  • Empty getRandom. The problem guarantees at least one element exists when called, so we don't need to defend against list.size() == 0.

Complexity

  • insert: O(1) average — one hashmap lookup + one append.
  • remove: O(1) average — one hashmap lookup, one swap, one tail pop, one map delete.
  • getRandom: O(1) — random index + one list lookup.
  • Space: O(n) total — the list and the map each hold at most one entry per value.
"Average" not "worst case." The O(1) bounds rely on the hashmap, so a degenerate hash that buckets many keys into the same chain could make individual operations O(n). That's the standard caveat; under Java's HashMap with reasonable inputs it holds in practice.
