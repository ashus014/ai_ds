2246. Longest Path With Different Adjacent Characters

You are given a tree on n nodes (parent array) and a string s of length n — character s[i] sits on node i. Find the length of the longest simple path in the tree such that no two adjacent nodes on the path share the same character. The winning pattern is the same “two best downward chains + update global answer at each node” structure as LeetCode 124 — Binary Tree Maximum Path Sum, except edges with matching characters are treated as having gain 0 (skip merging across that child).

Microsoft · Hard

Summary

Same core idea as LC 124: At each node, DFS returns the best downward path length ending at that node (toward the parent). While combining children, keep the top two child contributions that are still “valid” (child label ≠ current label). The global answer is the best either of: (a) extend only one branch through the node, or (b) stitch two branches through the node — 1 + longest + second_longest — mirroring left_gain + node + right_gain in 124.

Hint: Build adjacency from the parent array (treat the input as an undirected tree for DFS from root 0). For each child subtree, recurse first; if s[child] == s[curr], treat that subtree’s contribution as unusable for chaining through curr. Maintain two best heights among valid children, then relax result the same way you would for a diameter / max path sum.

Answer

✅ DFS — two best child chains (LC 124 analogue)

✅ Solution.java — two chains + global max 
import java.util.*;

class Solution {

    int result = 1;

    public int longestPath(int[] parent, String s) {
        int n = parent.length;

        // build graph (undirected like your C++ code)
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            adj.add(new ArrayList<>());
        }

        for (int i = 1; i < n; i++) {
            adj.get(parent[i]).add(i);
            adj.get(i).add(parent[i]);
        }

        DFS(adj, 0, -1, s);
        return result;
    }

    private int DFS(List<List<Integer>> adj, int curr, int parent, String s) {

        int longest = 0;
        int second_longest = 0;

        for (int child : adj.get(curr)) {

            if (child == parent) continue;

            int child_longest_length = DFS(adj, child, curr, s);

            // skip same character
            if (s.charAt(child) == s.charAt(curr)) continue;

            // same logic as your C++ snippet
            if (child_longest_length > second_longest)
                second_longest = child_longest_length;

            if (second_longest > longest) {
                int temp = longest;
                longest = second_longest;
                second_longest = temp;
            }
        }

        // ---------------- SAME AS LEETCODE - 124 ----------------

        int koi_ek_acha = Math.max(longest, second_longest) + 1;

        int only_root_acha = 1;

        int neeche_hi_milgaya_answer = 1 + longest + second_longest;

        result = Math.max(result,
                Math.max(koi_ek_acha,
                Math.max(only_root_acha, neeche_hi_milgaya_answer)));

        return Math.max(koi_ek_acha, only_root_acha);
    }
}
124 vs 2246. In 124, each child always contributes max(dfs(child), 0) toward the path through the node. Here, if the child’s character equals the current node’s character, that edge cannot appear on a valid path — so you continue and that branch contributes nothing to longest / second_longest. The “pick two best downward lengths and form a path through the current vertex” step is otherwise the same mental model.

Complexity

  • Time: O(n) — each undirected edge is traversed a constant number of times in the DFS tree rooted at 0.
  • Space: O(n) for the adjacency lists plus O(h) recursion stack depth (h ≤ n).
