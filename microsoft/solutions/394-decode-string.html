394. Decode String

Given an encoded string s with pattern k[encoded_string], decode it in one pass. Digits form the repeat count k; square brackets nest.

Microsoft · Medium — 14 April 2026

Summary

Hint: Keep two stacks — repeat counts and the StringBuilder before each [. On [, push k and the current chunk, then reset. On ], pop the count, append the inner string that many times to the outer builder.

Answers

✅ Two stacks

✅ Solution.java — stacks 
class Solution {
    public String decodeString(String s) {
        Stack<Integer> countStack = new Stack<>();
        Stack<StringBuilder> stringStack = new Stack<>();
        StringBuilder currentString = new StringBuilder();
        int k = 0;

        for(char ch : s.toCharArray()) {
            if(Character.isDigit(ch)) {
                k = k * 10 + (ch - '0');
            } else if(ch == '[') {
                countStack.push(k);
                stringStack.push(currentString);
                currentString = new StringBuilder();
                k = 0;
            } else if(ch == ']') {
                StringBuilder decodedString = stringStack.pop();
                int count = countStack.pop();
                for(int i = 0; i < count; i++) {
                    decodedString.append(currentString);
                }
                currentString = decodedString;
            } else {
                currentString.append(ch);
            }
        }
        return currentString.toString();

    }
}
One linear scan. Time O(n) for output size proportional to decoded length; space O(n) for stacks and builders.

Complexity

  • Time: O(n) over the input, plus work to build the result (output can be exponential in bracket depth in worst case).
  • Space: O(n) for the stacks and intermediate strings.
