1304. Find N Unique Integers Sum up to Zero

Return any array of n unique integers whose sum is 0. Freedom in construction → the cleanest answer is a symmetric pair sweep: ±1, ±2, … plus a lone 0 when n is odd.

Microsoft · Easy

Summary

Hint: Every +k cancels with -k. Emit pairs (1,-1), (2,-2), …; if n is odd, drop a 0 in the middle. If you ever need a single-pass alternative, place 1..n-1 and set the last slot to -(1+2+…+(n-1)).

Answers

Naive approach — fill 1..n-1, make last slot cancel the sum

Solution.java — fill & cancel 
class Solution {
    public int[] sumZero(int n) {
        int[] res = new int[n];
        int sum = 0;

        for (int i = 0; i < n - 1; i++) {
            res[i] = i + 1;   // 1,2,3...
            sum += res[i];
        }

        res[n - 1] = -sum;  // make total zero

        return res;
    }
}
Works, but the last element can grow quadratically with n (1+2+…+(n-1) = n(n-1)/2). Fine for the given constraints, but the symmetric version below is cleaner and bounded by n/2.

✅ Optimal approach — emit symmetric pairs ±i

✅ Solution.java — symmetric ±i 
class Solution {
    public int[] sumZero(int n) {
        int[] res = new int[n];
        int index = 0;

        for (int i = 1; i <= n / 2; i++) {
            res[index++] = i;
            res[index++] = -i;
        }

        // if n is odd, add 0
        if (n % 2 == 1) {
            res[index] = 0;
        }

        return res;
    }
}

Complexity

  • Time: O(n) — single pass to fill the result.
  • Space: O(1) extra (output array excluded).
  • Value range (optimal): every element lies in [-n/2, n/2] — no overflow, unique by construction.
