453. Minimum Moves to Equal Array Elements

In one move you may choose n − 1 elements and increment each by 1. That is equivalent to decrementing the single element you did not choose by 1 (relative to the others). The entire process only cares how far each value sits above the minimum — the answer is Σ (nums[i] − min) = sum(nums) − n · min(nums) in one pass.

Microsoft · Medium — 14 April 2026

Summary

Hint: Raising everyone except the max by 1 is the same game as lowering the max by 1 (think relative gaps). When all values meet at the minimum, the number of decrements of the max equals how much “excess” mass sat above min across the whole array.

Answers

Naive approach — simulate “increment all but max”

Solution.java — simulate +1 on n−1 
class Solution {
    public int minMoves(int[] nums) {
        int moves = 0;

        while (!allEqual(nums)) {
            int maxIndex = getMaxIndex(nums);

            for (int i = 0; i < nums.length; i++) {
                if (i != maxIndex) {
                    nums[i]++;
                }
            }
            moves++;
        }

        return moves;
    }

    private boolean allEqual(int[] nums) {
        for (int num : nums) {
            if (num != nums[0]) return false;
        }
        return true;
    }

    private int getMaxIndex(int[] nums) {
        int idx = 0;
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] > nums[idx]) {
                idx = i;
            }
        }
        return idx;
    }
}
Correct but each “round” can cost O(n) and you may need on the order of (max − min) rounds — huge when values are large. LeetCode constraints make this timeout.

Better naive — repeatedly decrement the current maximum

Solution.java — decrement max until flat 
class Solution {
    public int minMoves(int[] nums) {
        int moves = 0;

        while (true) {
            int maxIdx = getMaxIndex(nums);
            int min = getMin(nums);

            // if all equal → done
            if (nums[maxIdx] == min) break;

            nums[maxIdx]--; // reduce max
            moves++;
        }

        return moves;
    }

    private int getMaxIndex(int[] nums) {
        int idx = 0;
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] > nums[idx]) idx = i;
        }
        return idx;
    }

    private int getMin(int[] nums) {
        int min = nums[0];
        for (int num : nums) {
            min = Math.min(min, num);
        }
        return min;
    }
}
Same move count as the first simulation, but each move is O(n) to find the max (and min). Still O((max − min) · n) — too slow for large gaps.

✅ Optimal — closed form

Per element, the number of times it participates in an effective “pull toward min” is num − min. Summing:

  • (num₁ − min) + … + (numn − min) = sum(nums) − n · min

Example nums = [1, 2, 3], min = 1 → gaps 0 + 1 + 2 = 3.

✅ Two-pass: sum of (num − min)

✅ Solution.java — Σ(num − min) 
class Solution {
    public int minMoves(int[] nums) {
        int min = Integer.MAX_VALUE;

        // find minimum
        for (int num : nums) {
            min = Math.min(min, num);
        }

        int total = 0;

        // sum of (num - min)
        for (int num : nums) {
            total += (num - min);
        }

        return total;
    }
}

✅ One-pass: sum(nums) − n · min

✅ Solution.java — sum − n·min 
class Solution {
    public int minMoves(int[] nums) {
        int min = Integer.MAX_VALUE;
        int sum = 0;

        for (int num : nums) {
            min = Math.min(min, num);
            sum += num;
        }

        return sum - (min * nums.length);
    }
}

Complexity

  • Naive simulations: time exponential / pseudo-polynomial in (max − min) · n; space O(1) besides the array.
  • ✅ Optimal: time O(n), space O(1) — one or two linear passes, no moves simulated.

Comparison table

Approach Time Extra space
Simulate increment n−1 per move unbounded / pseudo-polynomial O(1)
Repeatedly decrement max O((max − min) · n) O(1)
