Python programming environment

  • Google Colab — online environment for Python, built on Jupyter Notebook.
  • Ready-to-use Python runtime with libraries such as NumPy, Pandas, and Matplotlib pre-installed.
  • Creating notebooks, running code cells, and using markdown cells for documentation.
  • Comments: # for single-line; multi-line commenting often done with Ctrl + / (toggle comment) in the editor.

Python vs other languages

  • Indentation (typically 4 spaces) defines blocks — not curly braces {}.
  • Statements end with new lines; semicolons are not required.
  • Dynamically typed — no explicit type declaration on variables.
  • No mandatory main() entry point (unlike C/Java).
  • Interpreted — code runs line by line.
  • Readable, almost English-like syntax; large ecosystem and community.

Variables and data types

  • Types are inferred from the assigned value (dynamic typing).
  • Duck typing: what matters is how an object behaves, not its declared type.
  • Variables can be reassigned to different types without error.
  • Internally, names refer to objects in memory (references), not raw value slots.
  • Garbage collection reclaims unused objects automatically.
  • Everything is an object; there are no separate “primitive” types like in some languages.

Basic data types explained

  • Integers: arbitrary precision in Python (unlike fixed 32/64-bit integers in many languages).
  • Floats: stored in binary — watch for precision quirks (e.g. 0.3 vs 0.30000004).
  • Booleans: subclass of int (True == 1, False == 0) — useful for counting and logic.
  • None: Python's null; prefer is None / is not None over == None.

Operators in Python

Seven broad categories: arithmetic, comparison, logical, bitwise, assignment, membership, and identity.

Arithmetic operators

  • In Python 3, / always returns a float (unlike Python 2's integer division for ints).
  • Precedence: exponentiation first, then multiplication/division, then addition/subtraction (standard math rules).
  • Augmented assignment: e.g. x += 5 combines operation and assignment.
  • Walrus operator := (Python 3.8+): assign and use a value in one expression.

Assignment operators

  • Simple: x = 10
  • Augmented: x += 5, x -= 3, x *= 2, etc.
  • Multiple assignment: a, b, c = 1, 2, 3
  • Swapping: a, b = b, a

Identity operators

Test whether two names refer to the same object in memory.

Operator Description Example Result
isSame objecta is bTrue if same object
is notNot the same objecta is not bTrue / False
  • is / is not → object identity (same address).
  • == / != → value equality.
  • Analogy: twins can look identical (==) but are not the same person (is).

Important concepts highlighted

  • Indentation defines code structure.
  • Dynamic typing and duck typing support flexible APIs.
  • Variables are references to objects.
  • Float precision: use tools like math.isclose when comparing floats in data work.
  • Use is None for null checks.
  • Short-circuit evaluation in and / or for efficiency and safety.
  • Bitwise ops are rarer in everyday scripts but matter in some numerical/low-level contexts.
  • Walrus operator := for inline assignment (3.8+).
  • Multiple assignment and tuple swapping for clear, compact code.
  • Comments and markdown cells document intent and workflow.
