Python Programming Lecture — Summary Notes

Lecture by Anjit Ranjan Sha (Assistant Professor, Mathematics; CS background). Overview of Python for beginners, with terminology and concepts for further study.

1. Motivation for learning Python

  • Central in AI, ML, and DL; many models are built with Python.
  • Systems like ChatGPT rest on deep learning stacks that are largely implemented in Python.
  • Broad use in education, industry, and research.
  • Versatile: web development, automation, data science, computer vision, image processing, and more.

2. Python history and versions

Year / era Version / feature Notes
1980 Development started Implemented in C; foundational runtime layer is C.
1991 Python 0.90 released Initial public version.
Early 2000s Python 2 Very popular; now deprecated (support ended ~2020).
Post-2020 Python 3.x Dominant line; ongoing releases (e.g. 3.14, 3.16) emphasise security among other improvements.
  • Python is open source and free.
  • Code is run by an interpreter, not compiled as one monolithic binary step like typical C workflows.

3. Execution model: interpreter vs compiler

  • C: compiler turns the whole program into machine code before run.
  • Python: interpreter runs line by line, translating and executing incrementally.
  • Easier debugging: failures surface at the line where they occur; earlier lines may already have run.
  • CPython is built on C; execution paths may involve translation toward C-level work — abstracted for everyday coding.
  • Trade-off: Python is typically slower than C, but faster to write, test, and maintain.

4. Python vs C: key differences

Feature Python C
Language type Interpreted; dynamically typed Compiled; statically typed
Syntax Close to English; beginner-friendly Lower-level, more verbose
Object orientation Supports OOP Procedural focus (no native OOP like Python's classes)
Pointers Not exposed; managed implicitly Explicit pointers
Typing Weakly typed, dynamic type inference (as presented in lecture) Strongly typed, fixed types after declaration
Variables Can rebind to different types Type fixed after declaration
Memory Automatic management Manual; pointer discipline required

5. Python language features and syntax

  • Interpreted, interactive, portable across Linux, Windows, and other platforms.
  • Supports procedural, object-oriented, and functional styles.
  • No separate type declarations — types are inferred at runtime.
  • Dynamic rebinding (e.g. month = "May" then month = 9).
  • Case-sensitive names (x vs X).
  • Identifiers: letters, digits, underscore; must not start with a digit.
  • Multiple assignment in one line, e.g. a, b, c = 1, 2, "Python".

6. Data types and collections

Category Examples & description
Numeric Integer, float, complex, boolean
Sequences Strings; lists (mutable, mixed types); tuples (immutable sequences)
Sets Unordered; unique elements only (duplicates dropped)
Dictionaries Key–value maps; keys unique; values any type — fast lookup, e.g. d['name'] = "Zone"
  • Lists allow mixed element types.
  • Tuples behave like fixed lists (immutable).
  • Sets enforce uniqueness automatically.

7. Type conversion and dynamic typing

  • Implicit and explicit conversions both arise in practice.
  • int() — truncates toward zero (not “round”).
  • float(), str() — common explicit casts.
  • String → float → int chains must be staged; direct invalid paths raise errors.
  • Important in ML/AI pipelines where data is often normalised to floats for numerics.

8. Use cases, careers, and ecosystem

  • Data science and scientific computing.
  • Frameworks: TensorFlow, PyTorch, Keras.
  • Computer vision (e.g. perception stacks in autonomous systems).
  • Web development and automation.
  • Roles: developer, data scientist, ML engineer, and related paths.
  • Libraries such as NumPy (arrays/linear algebra), Pandas (tables/ETL), Matplotlib (plotting) extend the core language.

Core concepts and insights

  • Line-by-line interpretation supports rapid iteration and debugging.
  • Dynamic typing and readable syntax help beginners and prototypes.
  • Speed vs C is offset by development velocity and ecosystem depth.
  • Hiding pointers reduces low-level mistakes.
  • Strong presence in AI/ML makes Python a practical career foundation.
  • Collections and conversions show up constantly in real code.
