Why Python?

Python is used almost everywhere — especially in ML, AI, and deep learning. Every model like ChatGPT's GPT (a transformer) is built using Python code.

Areas where Python is used:

  • Machine Learning & Deep Learning
  • Data Science & Scientific Computing
  • Web Development
  • Computer Vision
  • Image Processing
  • Automation
  • Research & Education

Brief History

  • Started in 1980, first version (0.90) released in 1991
  • Python 2 was popular but officially discontinued around 2020
  • Today only Python 3 is actively maintained (latest: 3.14+)
  • Recent versions (3.14/3.16) are adding stronger focus on security

How Python Works — Compiled vs Interpreted

C (Compiled)

  • You write the full program first
  • Compiler converts everything to binary all at once
  • Any error = entire program fails to compile
  • Faster execution

Python (Interpreted)

  • Executes line by line
  • Interpreter converts one line → runs it → moves to next line
  • Error stops execution at that line; earlier lines already ran
  • Slower, but far easier to debug and write

Under the hood

Python is written in C. When Python code runs, it internally goes:
Python → C-equivalent → binary
This multi-step process is why Python is slower than C, but the developer experience is much simpler.

Python vs C — Key Differences

Feature Python C
Speed Slower (multi-step translation) Faster
Pointers Handled implicitly (no headache) Explicit pointers
Typing Dynamic — type inferred, can change Static — declare once, fixed
OOP Supported Not supported (procedural only)
Memory Managed automatically Manual control
Portability Cross-platform Cross-platform

Core Syntax Basics

Printing output

print("Hello World")

Variables — no type declaration needed

month = "May"   # string — Python decides the type
age = 18        # integer
price = 99.5    # float

Python infers the type from the assigned value. The same variable can be reassigned a different type later (dynamic typing).

Multiple assignment in one line

a = b = c = 10              # same value for all
x, y, z = 1, 2, "python"   # different values

Checking type

type(month)  # <class 'str'>
type(age)    # <class 'int'>
type(price)  # <class 'float'>

Type conversion

int(3.9)    # → 3       (decimal part removed, NOT rounded)
float(10)   # → 10.0
str(25)     # → "25"
Note: Converting "3.9" (string) directly to int raises a ValueError.
Do it in two steps: int(float("3.9")) → 3

Naming Rules

  • Can contain letters, digits, and underscores
  • Cannot start with a digit (e.g. 9x is invalid)
  • No special characters except underscore
  • Case-sensitive — x and X are two different variables

Data Types in Python

Numeric types

  • int — integers (e.g. 10, 18)
  • float — decimal numbers (e.g. 99.5, 10.0)
  • complex — complex numbers
  • bool — True / False

Collection / Sequence types

List — ordered, mutable, allows mixed types and duplicates

my_list = [1, 2, "hello", 10.0]
my_list[2] = 99   # you can change values

Tuple — like a list but immutable (values cannot be changed after creation)

my_tuple = (1, 2, 3)
my_tuple[0] = 99  # ❌ raises TypeError

Set — unordered, only unique elements (duplicates auto-removed)

my_set = {1, 2, 2, 3}  # → {1, 2, 3}

Dictionary — key-value pairs; access value by key

d = {"name": "Zon", "age": 20}
d["name"]   # → "Zon"
d["age"]    # → 20
d.keys()    # → dict_keys(['name', 'age'])

Key Features of Python

  • Easy to read, write, and debug
  • Open source and free
  • Beginner-friendly syntax (close to English)
  • Supports object-oriented programming
  • Dynamically typed
  • Large standard library + thousands of third-party packages
  • Portable across Linux, Windows, macOS
