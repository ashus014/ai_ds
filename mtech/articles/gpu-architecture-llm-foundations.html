GPU architecture vs CPU — Foundations for deep learning & LLMs

Parallelism, memory hierarchy, and why GPUs power modern AI

📅 March 22, 2026 · Large language models (LLM) · IIT Patna AI & DS

Summary

This article gives an overview of GPU architecture and its distinction from CPUs, emphasizing their critical role in modern computing tasks like video editing, deep learning, and training large language models (LLMs). It is meant to give software engineers foundational GPU knowledge so they can better understand and build AI/ML models.

Key topics covered

  • Comparison between CPUs and GPUs
  • Evolution of GPU technology
  • GPU components and memory hierarchy
  • Parallel processing and throughput
  • Applications in video editing and deep learning
  • Memory optimization techniques like Flash Attention

Core concepts and highlights

CPU vs GPU architecture and design goals

CPU:

  • Designed for sequential instruction execution with low latency.
  • Emphasizes reducing instruction latency via large caches and control units.
  • Contains a small number of powerful ALUs (Arithmetic Logic Units).
  • Ideal for complex single tasks requiring precision.
  • Cache hierarchy:
    • L1 (fastest, smallest)
    • L2 (larger, slower)
    • L3 (largest, slowest) caches to minimize RAM access latency.

GPU:

  • Designed for massive parallelism and high throughput.
  • Contains many simpler ALUs (cores) optimized for repetitive, similar tasks.
  • Uses small cache and control units to maximize chip space for cores.
  • Excels in parallel processing, handling millions or billions of computations simultaneously.
  • Specialized hardware like CUDA cores for general computation and Tensor cores for deep learning acceleration.
  • Can tolerate higher instruction latency by switching between many threads, ensuring maximum utilization of compute units.
Feature CPU GPU
Primary goal Low instruction latency High throughput
ALUs Fewer, more powerful Many, simpler
Cache size Large (L1, L2, L3 caches) Small caches
Control units Many Few
Parallelism Low (few cores) Very high (hundreds/thousands of cores)
Typical use case Sequential, complex tasks Parallel, repetitive tasks

Memory hierarchy and data flow

  • Data flows from storage devices (HDD/SSD) → RAM → CPU/GPU caches → cores.
  • GPUs have dedicated memory called VRAM (Video RAM), which stores data actively being processed.
  • GPU memory hierarchy:
    • VRAM (large capacity, e.g. 16GB or 40GB depending on model)
    • L2 cache (holds frequently used data from VRAM)
    • Streaming Multiprocessors (SMs) with their own L1 cache / Shared SRAM
  • Specialized GPU components include CUDA cores and Tensor cores:
    • CUDA cores: general-purpose calculations.
    • Tensor cores: optimized for deep learning matrix operations.

GPU parallelism and latency tolerance

  • GPUs handle parallel workloads by scheduling thousands of threads simultaneously.
  • When some threads wait for data (high latency), the GPU switches to others, maintaining high utilization.
  • This thread-level parallelism allows GPUs to achieve high throughput despite higher instruction latency per thread.

Application in video editing and deep learning

  • GPUs accelerate complex video editing tasks such as applying filters and effects by processing many pixels or frames simultaneously.
  • Their parallel processing power and specialized hardware make GPUs ideal for machine learning workloads, especially training deep neural networks.
  • Tensor cores significantly speed up deep learning computations by optimizing matrix multiplications.
  • GPUs use High Bandwidth Memory (HBM) to boost data transfer rates between memory and processors, improving training efficiency.

Advanced optimization: Flash Attention

  • Flash Attention is an optimization technique to maximize GPU memory utilization.
  • It increases the speed of data movement between GPU memory layers like VRAM, L2 cache, and L1 cache/SRAM.
  • Combines multiple small GPU tasks (kernels) into one larger task (kernel fusion), reducing overhead.
  • This technique unlocks the full potential of GPU parallelism and memory efficiency, critical for training large language models.

Key insights

  • GPUs are optimized for throughput via parallelism, contrasting CPUs’ low-latency sequential execution.
  • The large number of GPU cores and threads enable efficient handling of massive data simultaneously—a critical factor in AI and video rendering.
  • Memory hierarchy and specialized cores (CUDA and Tensor) are essential for GPU performance in both graphics and deep learning domains.
  • Techniques like Flash Attention and kernel fusion are vital for optimizing GPU usage in training large AI models.
  • Understanding these fundamentals provides a foundation for building efficient AI/ML models and leveraging GPU power effectively.

Conclusion

Taken together, these ideas demystify GPU architecture, contrast it with CPU design, and highlight why GPUs are indispensable for modern AI and video processing tasks. By grasping GPU parallelism, memory hierarchy, and specialized hardware, software engineers can better harness GPU capabilities in building scalable large language models and complex AI applications.
