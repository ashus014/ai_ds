Session focus

This session covers print() and input(), integer caching, string formatting (especially f-strings), type casting, and reading multiple values from one line. The emphasis is on how Python simplifies I/O compared with languages like C, C++, and Java.

Big idea: Flexible printing, disciplined casting for input(), and f-strings for readable output — with string methods to clean user data.

The print() function

  • print() can output strings, numbers (int/float), booleans, and expressions.
  • String literals may use single '...' or double "..." quotes; do not mix quote styles inside one string.

Integer caching in CPython

  • CPython caches small integers in the range −5 through 256 for memory and speed.
  • Names bound to the same cached value can share the same object: id(x) == id(y) may be True for e.g. x = 100 and y = 100.
  • Outside that range, or for floats, you generally get distinct objects — do not rely on is for value equality.
Remember: use == for values; reserve is for identity (e.g. None, singletons).

input() and type casting

  • input() always returns a str.
  • Cast before arithmetic: age = int(input("Enter age: ")), weight = float(input("Enter weight: ")).
  • Skipping casts causes type errors when you add or multiply with numbers.

Multiple inputs and splitting

  • Read one line (e.g. comma-separated), then .split(",") or .split() (whitespace).
  • Use .strip() on each piece to remove surrounding spaces.
  • Convert numeric fields with int() / float().

Printing multiple values

  • Multiple arguments: print(a, b, c) — default sep=" " (space).
  • sep customises the delimiter between items.
  • end defaults to newline "\n"; use e.g. end="" or end="..." to continue on the same line.

F-strings (formatted string literals)

  • Preferred modern style: print(f"My name is {name}, age is {age}").
  • Expressions inside { } are evaluated and inserted.
  • Common format specs: {x:.2f} (float precision), {n:,} (thousands separators), alignment < / > / ^ with width, {n:05d} (zero-pad integers), {x:e} (scientific notation).

Other formatting styles

  • "{} {}".format(a, b) — .format() with placeholders.
  • "%s %d" % (s, n) — C-style % formatting (legacy).

Type conversion functions

Function Purpose
int()Convert to integer
float()Convert to floating-point
str()Convert to string
bool()Convert to boolean

bool(): 0, empty containers, empty strings, and None are falsy; most other values are truthy.

Strings and string methods

  • Strings are immutable; methods return new strings.
  • .strip() — trim leading/trailing whitespace.
  • .lower() / .upper() / .title().
  • Length: len(s) (built-in, not a method).
  • Concatenation with + or, preferably, f-strings.

Procedural vs object-oriented view

  • Python supports a hybrid style: procedural scripts and OOP together.
  • Methods are functions tied to objects (e.g. str.upper).
  • Built-ins like print() and len() are not methods on a particular class instance in the usual sense.

Code examples

Copy each snippet from its window toolbar (same style as other notes).

Print: multiple arguments and sep

example_print_sep.py 
print("Name:", "Alice", "Age:", 25)       # default space between items
print("Alice", "Bob", "Charlie", sep=", ")  # comma+space between items

Print: custom end

example_print_end.py 
print("Loading", end="...")
print("Done")   # → Loading...Done (same line)

F-string with formatting

example_fstring.py 
name = "Alice"
age = 28
salary = 87500
print(f"My name is {name}, age is {age}, salary is ${salary:,}")

Input and cast

example_input_cast.py 
age = int(input("Enter your age: "))
print(age + 1)

Split comma-separated input

example_split_csv.py 
raw = input("Enter name, age, city: ")
parts = raw.split(",")
name = parts[0].strip()
age = int(parts[1].strip())
city = parts[2].strip()
print(f"Name: {name}, Age: {age}, City: {city}")

Several numbers on one line

example_split_numbers.py 
numbers = input("Enter three numbers: ").split()
print(int(numbers[0]) + 1, int(numbers[1]) + 1, int(numbers[2]) + 1)
