714. Best Time to Buy and Sell Stock with Transaction Fee

LeetCode 75 #65 — Unlimited transactions, but each sale pays a fixed fee fee. State buy = 1 means you may buy (not holding); buy = 0 means you hold one share and may sell. Recurse day by day: buy today, skip, sell today, skip—take max profit.

LeetCode 75 / DP — Multidimensional

Hint

Subproblem is (day, buy). Selling adds prices[day] - fee and flips back to “can buy”. Memo dimensions n × 2.

Answers

Naive recursion

DFS on day and whether you are allowed to buy; try take / skip each action.

Naive — reference only 
class Solution {
    int FEE;

    public int maxProfit(int[] prices, int fee) {
        FEE = fee;
        int n = prices.length;

        // Start from day 0, and initially we are allowed to BUY
        return maxP(prices, 0, n, 1);
    }

    int maxP(int[] prices, int day, int n, int buy) {
        // Base case → no days left → no profit
        if (day >= n) return 0;

        int profit = 0;

        // Case 1: We are allowed to BUY
        if (buy == 1) {

            int consider = maxP(prices, day + 1, n, 0) - prices[day];
            int notConsider = maxP(prices, day + 1, n, 1);

            profit = Math.max(profit, Math.max(consider, notConsider));
        } 
        
        // Case 2: We can SELL
        else {

            int consider = maxP(prices, day + 1, n, 1) + prices[day] - FEE;
            int notConsider = maxP(prices, day + 1, n, 0);

            profit = Math.max(profit, Math.max(consider, notConsider));
        }

        return profit;
    }
}
Naive approach. The pair (day, buy) takes only O(n) values, but recursion revisits each state exponentially often. Memoization or a 2D DP table reduces this to O(n) time.

✅ Memoization (top-down DP)

Cache maxP(day, buy) in t[day][buy]. Table size 50001 × 2 matches your field; only rows 0 … n−1 are initialized to -1 before the search.

Solution.java — memoization 
class Solution {
    // t[day][buy] → memo table
    // day = current index
    // buy = 1 → we are allowed to BUY
    // buy = 0 → we are holding a stock, so we can SELL
    int[][] t = new int[50001][2];

    // transaction fee (global so we don’t pass every time)
    int FEE;

    public int maxProfit(int[] prices, int fee) {
        FEE = fee;
        int n = prices.length;

        // Initialize memo table with -1 (means not computed yet)
        for (int i = 0; i < n; i++) {
            t[i][0] = -1;
            t[i][1] = -1;
        }

        // Start from day 0, and initially we are allowed to BUY
        return maxP(prices, 0, n, 1);
    }

    int maxP(int[] prices, int day, int n, int buy) {
        // Base case → no days left → no profit
        if (day >= n) return 0;

        // If already computed, return stored result
        if (t[day][buy] != -1) return t[day][buy];

        int profit = 0;

        // Case 1: We are allowed to BUY
        if (buy == 1) {

            // Option 1: BUY stock today
            // → spend money (prices[day])
            // → next state becomes SELL (buy = 0)
            int consider = maxP(prices, day + 1, n, 0) - prices[day];

            // Option 2: SKIP buying today
            // → stay in BUY state
            int notConsider = maxP(prices, day + 1, n, 1);

            // Take max profit of both choices
            profit = Math.max(profit, Math.max(consider, notConsider));
        } 
        
        // Case 2: We have already bought, so we can SELL
        else {

            // Option 1: SELL stock today
            // → earn money (prices[day])
            // → pay transaction fee
            // → next state becomes BUY again
            int consider = maxP(prices, day + 1, n, 1) + prices[day] - FEE;

            // Option 2: SKIP selling today
            // → stay in SELL state (holding stock)
            int notConsider = maxP(prices, day + 1, n, 0);

            // Take max profit of both choices
            profit = Math.max(profit, Math.max(consider, notConsider));
        }

        // Store result in DP table and return
        return t[day][buy] = profit;
    }
}
