72. Edit Distance

LeetCode 75 #66 — Minimum operations to turn s1 into s2 using insert, delete, or replace on one character at a time. Align suffixes with indices (i, j): end of s1 → insert rest of s2; end of s2 → delete rest of s1; match → advance both; else 1 + min(insert, delete, replace).

LeetCode 75 / DP — Multidimensional

Hint

State is (i, j). Cache solve(i, j) in t[i][j]. Table size can be (m+1)×(n+1); your fixed 501×501 fits LeetCode constraints.

Answers

Naive recursion

Explore insert / delete / replace without memo.

Naive — reference only 
class Solution {
    int m, n;

    public int minDistance(String s1, String s2) {
        m = s1.length();
        n = s2.length();

        return solve(s1, s2, 0, 0);     
    }

    private int solve(String s1, String s2, int i, int j) {

        // If s1 finished → insert remaining of s2
        if(i == m) { 
            return n - j; 
        } 
        
        // If s2 finished → delete remaining of s1
        else if(j == n) {
            return m - i; 
        }

        // If characters match → move both
        if(s1.charAt(i) == s2.charAt(j)) {
            return solve(s1, s2, i + 1, j + 1);
        } 

        // Try all 3 operations
        int insert = 1 + solve(s1, s2, i, j + 1);
        int delete = 1 + solve(s1, s2, i + 1, j);
        int replace = 1 + solve(s1, s2, i + 1, j + 1);

        return Math.min(insert, Math.min(delete, replace));
    }
}
Naive approach. There are only O(m·n) pairs (i, j), but each is recomputed many times, so runtime grows exponentially without memoization or a bottom-up table.

✅ Memoization (top-down DP)

Reuse the same recurrence; store solve(i, j) in t[i][j] after the first compute.

Solution.java — memoization 
class Solution {
    int m, n;

    // DP table → t[i][j] stores answer for:
    // "minimum operations to convert s1[i..end] → s2[j..end]"
    int[][] t= new int[501][501];

    public int minDistance(String s1, String s2) {
        m = s1.length();
        n = s2.length();

        // Initialize DP table with -1 (means not computed yet)
        for (int i = 0; i < 501; i++) {
            for (int j = 0; j < 501; j++) {
                t[i][j] = -1;
            }
        }

        // Start comparing from index 0 of both strings
        return solve(s1, s2, 0, 0);     
    }

    private int solve(String s1, String s2, int i, int j) {

        // 🔴 Base Case 1:
        // If s1 is finished → we need to INSERT remaining characters of s2
        if(i == m) { 
            return n - j; 
        } 
        
        // 🔴 Base Case 2:
        // If s2 is finished → we need to DELETE remaining characters of s1
        else if(j == n) {
            return m - i; 
        }

        // If already computed → return from cache
        if(t[i][j] != -1) {
            return t[i][j];
        }

        // 🟢 If characters match → no operation needed
        if(s1.charAt(i) == s2.charAt(j)) {
            // Move both pointers forward
            return t[i][j] = solve(s1, s2, i + 1, j + 1);
        } 

        // 🔵 If characters don't match → we have 3 choices:

        // 1. INSERT → insert s2[j] into s1
        // (i stays same, j moves forward)
        int insert = 1 + solve(s1, s2, i, j + 1);

        // 2. DELETE → delete s1[i]
        // (i moves forward, j stays same)
        int delete = 1 + solve(s1, s2, i + 1, j);

        // 3. REPLACE → replace s1[i] with s2[j]
        // (both move forward)
        int replace = 1 + solve(s1, s2, i + 1, j + 1);

        // Take minimum of all operations
        return t[i][j] = Math.min(insert, Math.min(delete, replace));
    }
}
