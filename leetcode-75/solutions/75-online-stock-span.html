901. Online Stock Span

LeetCode 75 #75 — On each next(price), return the span of consecutive days (including today) with price <= today’s price ending at today. A list plus backward scan is simple but quadratic overall; a stack of [price, span] merges “dominated” days in amortized constant time per call.

LeetCode 75 / Monotonic Stack

Hint

Brute: append the price, walk backward while previous prices are <= current. Stack: while top price <= current, absorb its span into yours and pop; then push {price, span}.

Answers

Brute force (list + backward scan)

Keep every price in an ArrayList, then count backward from yesterday.

StockSpanner.java — brute force 
class StockSpanner {

    // Store all prices seen so far
    private List<Integer> prices;

    public StockSpanner() {
        prices = new ArrayList<>();
    }
    
    public int next(int price) {

        // Add current price
        prices.add(price);

        int span = 1;

        // Start from previous day and go backwards
        int i = prices.size() - 2;

        while(i >= 0 && prices.get(i) <= price) {
            span++;
            i--;
        }

        return span;
    }
}
Complexity. Day k may scan k older entries, so over n days the total work is on the order of 1 + 2 + … + n = O(n²). Add import java.util.List; and import java.util.ArrayList; in a full Java file.

✅ Stack (price, span)

Each stack entry summarizes a contiguous block of days that are already “covered” by a single span at that price. Popping merges strictly lower-or-equal prices into the current span.

StockSpanner.java — monotonic stack 
class StockSpanner {

    private Stack<int[]> stack;

    public StockSpanner() {
        stack = new Stack<>();    
    }
    
    public int next(int price) {
        int span = 1;
        while(!stack.isEmpty() && stack.peek()[0] <= price) {
            span += stack.pop()[1];
        }

        stack.push(new int[]{price, span});
        return span;
    }
}

/**
 * Your StockSpanner object will be instantiated and called as such:
 * StockSpanner obj = new StockSpanner();
 * int param_1 = obj.next(price);
 */
Java imports. Add import java.util.Stack; (and optionally drop unused imports). Each int[] holds [price, span] for a compressed group of days.

Why the stack solution is O(1) amortized per next

What you might think

That inner while can look scary: “many pops in one call → isn’t that O(n) every time?” Here is the loop:

while (!stack.isEmpty() && stack.peek()[0] <= price) {
    span += stack.pop()[1];
}

That loop can run many times on one call, so it feels like O(n) per next, and over n calls maybe O(n²)? That worst case for a single call can indeed be linear in stack size — but the story over the whole stream is different.

Actual picture: amortized O(1) total work

Key observation. Each stored pair is pushed once and popped at most once. When something is popped, it never comes back — it cannot be part of a later while again.

Total work over n prices.

  • Pushes: at most n (one new [price, span] per next).
  • Pops: at most n total across the entire run (each pop discards one distinct entry forever).
Push  → at most n
Pop   → at most n
-------------------
Total → 2n  →  O(n) overall

Per operation (amortized). Spread that O(n) total work across n calls: O(n) / n = O(1) amortized per next. “Amortized” means the average cost over the full sequence is constant, even though one isolated call might still pop many items.
