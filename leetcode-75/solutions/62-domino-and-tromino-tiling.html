790. Domino and Tromino Tiling

LeetCode 75 #62 — Count tilings of a 2 × n grid with 2 × 1 dominoes and L-shaped trominoes, modulo 109+7. This problem is formula-driven: once you know the linear recurrence, implementation is memoization or tabulation.

LeetCode 75 / DP — 1D

Hint

Let f(n) be the number of complete tilings of width n. Small n can be counted by hand to seed DP; for larger n, use the closed recurrence below and take everything mod M.

Theory — deriving f(n) = 2f(n−1) + f(n−3) like a homework write-up

Step 0 — notation. Let f(n) be the number of complete tilings of a 2 × n board using 2 × 1 dominoes and L-shaped trominoes. All arithmetic below is eventually taken modulo M = 109 + 7.

Step 1 — warm-up with small n. Draw the boards for n = 1, 2, 3 and count tilings. You should get

f(1) = 1,  f(2) = 2,  f(3) = 5.

Optionally define f(0) = 1 as “one way to tile an empty board” (empty product in combinatorics). That convention is only a bookkeeping device for the next step; your Java seeds 1, 2, 5 directly.

Step 2 — a “long” recurrence (expanded form). Do a careful case split on how the tiling can end at the right (or equivalently, how you first “close” the last columns). One standard decomposition proves that for every n ≥ 3, with f(0) = 1,

f(n) = f(n − 1) + f(n − 2) + 2 · (f(0) + f(1) + … + f(n − 3)).   (★)

Reading (★) in words: the first two terms are the usual “add a vertical domino at the end” and “cap the end with two horizontal dominoes” pictures. The big sum captures all ways a tromino (or a chain of forced tromino/domino patterns) leaves an unfinished “notch” that is finally closed only after stepping back through widths 0, 1, …, n − 3; the factor 2 is the mirror symmetry (two orientations of that situation). You do not need to memorize the picture for the algebra below—only that (★) is a proved identity in standard write-ups of this problem.

Step 3 — write the same identity for n − 1. If n − 1 ≥ 3, replace n by n − 1 in (★):

f(n − 1) = f(n − 2) + f(n − 3) + 2 · (f(0) + f(1) + … + f(n − 4)).   (★′)

Step 4 — subtract, and watch the sum telescope. Compute the left-hand difference f(n) − f(n − 1) using (★) minus (★′), grouping term by term:

LHS = f(n) − f(n − 1)
    = [f(n−1) − f(n−2)] + [f(n−2) − f(n−3)] + 2 · [ (f(0)+…+f(n−3)) − (f(0)+…+f(n−4)) ].

The bracket of sums is not mysterious: everything from f(0) up to f(n − 4) appears twice and cancels, leaving a single f(n − 3) inside. The two f(n − 2) terms also cancel. So

f(n) − f(n − 1) = f(n − 1) − f(n − 3) + 2 · f(n − 3) = f(n − 1) + f(n − 3).

Step 5 — rearrange to the short recurrence. Add f(n − 1) to both sides of f(n) − f(n − 1) = f(n − 1) + f(n − 3):

f(n) = 2 · f(n − 1) + f(n − 3),   n ≥ 4.

That is exactly what you implement: no more long tail in code—the subtraction in Step 4 already absorbed it.

Step 6 — bases for the program. Hardcode the three values you counted in Step 1. For n ≥ 4, apply the formula with % M after each multiplication/addition as in your solutions.

Answers

✅ Memoization (top-down)

Translate the formula directly: cache solve(n) in t[n].

Solution.java — memoization 
class Solution {
    int[] t;
    int M = 1000000007;
    public int numTilings(int n) {
        t = new int[n + 1];
        Arrays.fill(t, -1);
        return solve(n);
    }

    private int solve(int n) {
        if(n == 1 || n == 2) {
            return n;
        }

          if(n == 3) {
            return 5;
        }

        if(t[n] != -1) {
            return t[n];
        }
      
        return t[n] = (2 * solve(n - 1) % M + solve(n - 3) % M) % M;
    }
}

Tabulation (bottom-up)

Same recurrence, filled iteratively from the bases—no recursion stack.

Solution.java — tabulation 
class Solution {

    int M = 1000000007;
    public int numTilings(int n) {

        if(n == 1 || n == 2)
            return n;

        int[] dp = new int[n+1];
        dp[1] = 1;
        dp[2] = 2;
        dp[3] = 5;

        for(int i = 4; i <=n; i++) {
            dp[i] = (2 * dp[i - 1]%M + dp[i - 3]%M)%M;
        }

        return dp[n];
    }
}
