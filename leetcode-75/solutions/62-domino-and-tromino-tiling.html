790. Domino and Tromino Tiling

LeetCode 75 #62 — Count tilings of a 2 × n grid with 2 × 1 dominoes and L-shaped trominoes, modulo 109+7. This problem is formula-driven: once you know the linear recurrence, implementation is memoization or tabulation.

LeetCode 75 / DP — 1D

Hint

Let f(n) be the number of complete tilings of width n. Small n can be counted by hand to seed DP; for larger n, use the closed recurrence below and take everything mod M.

Theory — from expanded form to f(n) = 2f(n−1) + f(n−3)

Let f(n) denote the number of tilings of a 2 × n board. If you break the problem down by how the right boundary is finished, combinatorics on partial fronts often produces a relation that still couples f(n) to several earlier widths at once.

A fuller “expanded” picture (before any collapsing) looks symbolically like building f(n) from f(n−1), f(n−2), and then a tail of terms such as 2f(n−3), 2f(n−4), …—the same pattern shifted appears inside f(n−1) when you write it the same way. So you are in the situation you sketched: something in the spirit of f(n) = f(n−1) + f(n−2) + 2f(n−3) + 2f(n−4) + …, with similar tails for f(n−1), f(n−2), etc.

The key step is to subtract the appropriately scaled equation for f(n−1) from the equation for f(n). The long tail of f(n−4), f(n−5), … appears in both expanded expressions with matching coefficients, so those pieces cancel (telescoping) instead of being carried forever in code.

What survives is a short linear recurrence with constant coefficients:

f(n) = 2·f(n − 1) + f(n − 3)   (for n ≥ 4)

So the whole board count depends only on the previous width and the width three steps back. The factor 2 comes from how the two dominant extensions of a width-(n−1) tiling interact with domino/tromino endings; the f(n−3) term is the part of the geometry that does not fold into that 2f(n−1) lump and must be carried explicitly.

Base cases (by enumerating small boards): f(1) = 1, f(2) = 2, f(3) = 5. The recurrence applies from n = 4 onward. Every sum and product should be reduced modulo M = 1_000_000_007 to match the statement.

Answers

✅ Memoization (top-down)

Translate the formula directly: cache solve(n) in t[n].

Solution.java — memoization 
class Solution {
    int[] t;
    int M = 1000000007;
    public int numTilings(int n) {
        t = new int[n + 1];
        Arrays.fill(t, -1);
        return solve(n);
    }

    private int solve(int n) {
        if(n == 1 || n == 2) {
            return n;
        }

          if(n == 3) {
            return 5;
        }

        if(t[n] != -1) {
            return t[n];
        }
      
        return t[n] = (2 * solve(n - 1) % M + solve(n - 3) % M) % M;
    }
}

Tabulation (bottom-up)

Same recurrence, filled iteratively from the bases—no recursion stack.

Solution.java — tabulation 
class Solution {

    int M = 1000000007;
    public int numTilings(int n) {

        if(n == 1 || n == 2)
            return n;

        int[] dp = new int[n+1];
        dp[1] = 1;
        dp[2] = 2;
        dp[3] = 5;

        for(int i = 4; i <=n; i++) {
            dp[i] = (2 * dp[i - 1]%M + dp[i - 3]%M)%M;
        }

        return dp[n];
    }
}
