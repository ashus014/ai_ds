LFU Cache

Least Frequently Used cache: evict the key with the smallest use count. When several keys share that count, evict the least recently used among them (LFU with LRU tie-break).

LLD / Java — 12 April 2026

Summary

Track key → Node and group nodes by frequency. Each frequency bucket is its own doubly linked list (insert at head = MRU within that bucket; evict from tail at minFreq when over capacity). A get or put bumps the node’s frequency and may advance minFreq when the old minimum bucket empties.

Answers

✅ O(1) — frequency lists + LRU per frequency

Map<Integer, Node> for membership; Map<Integer, DLinkedList> for each frequency level; minFreq points at the eviction bucket.

  • get / put: constant-time hash + splice in a fixed number of DLL heads/tails.
  • Space: O(capacity) for nodes plus frequency structures.
Implementation. Annotated Java (with copy-to-clipboard windows like the LRU note) will be added here soon.
Eviction. Always remove the tail of the list at minFreq — that is the LRU among all keys tied for lowest frequency.

Complexity

  • get(key), put(key, value): O(1) time (hash + constant splices).
  • Space: O(capacity) for nodes and frequency lists.
