LRU Cache

Low-Level Design — least recently used cache with list-based naive implementation

12 April 2026

Summary

An LRU (Least Recently Used) cache stores key-value pairs with a fixed capacity. When full, it evicts the least recently used entry before inserting a new one. In this naive version, recency is maintained in a list by moving recently accessed or updated items to the front.

Naive Approach

  • Use a list of [key, value] pairs.
  • Keep most recently used entry at index 0.
  • For get(key), linearly scan the list; if found, move entry to front.
  • For put(key, value), update existing key by moving to front, or insert new key at front.
  • If capacity is full, remove the last element (least recently used).

LRUCache.java (naive approach) 
import java.util.*;

class LRUCache {

    List<int[]> cache;
    int n;

    public LRUCache(int capacity) {
        this.n = capacity;
        this.cache = new ArrayList<>();
    }

    public int get(int key) {

        for (int i = 0; i < cache.size(); i++) {
            if (cache.get(i)[0] == key) {

                int value = cache.get(i)[1];

                // move to front (most recent)
                cache.remove(i);
                cache.add(0, new int[]{key, value});

                return value;
            }
        }

        return -1;
    }

    public void put(int key, int value) {

        // check if key already exists
        for (int i = 0; i < cache.size(); i++) {
            if (cache.get(i)[0] == key) {

                // remove old position
                cache.remove(i);

                // insert at front (most recent)
                cache.add(0, new int[]{key, value});
                return;
            }
        }

        // if capacity full → remove least recently used (last element)
        if (cache.size() == n) {
            cache.remove(cache.size() - 1);
        }

        // insert new at front
        cache.add(0, new int[]{key, value});
    }
}

Complexity

  • get(key): O(n) due to linear search and potential shift during insert/remove.
  • put(key, value): O(n) due to linear search and list operations.
  • Space: O(capacity).
