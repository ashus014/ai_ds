2357. Make Array Zero by Subtracting Equal Amounts

Each operation subtracts the same positive value from every positive entry. The number of operations equals the number of distinct positive values in the array. Problem on LeetCode →

Amazon · Easy — 12 April 2026

Summary

Repeatedly subtract the current minimum positive value; each round clears one “level” of values. That is equivalent to counting how many different positive numbers appear — each unique value triggers one round before it becomes tied for minimum and is eliminated with its layer.

Answers

⚠️ Naive (simulate rounds)

Find min among positives, subtract it from all positives, increment operations, repeat until no positives remain.

⚠️ Solution.java — naive 
class Solution {
    public int minimumOperations(int[] nums) {
        int operations = 0;

        while (true) {
            int min = Integer.MAX_VALUE;

            // Step 1: find smallest non-zero
            for (int num : nums) {
                if (num > 0) {
                    min = Math.min(min, num);
                }
            }

            // If no non-zero element → done
            if (min == Integer.MAX_VALUE) {
                break;
            }

            // Step 2: subtract min from all positive elements
            for (int i = 0; i < nums.length; i++) {
                if (nums[i] > 0) {
                    nums[i] -= min;
                }
            }

            operations++;
        }

        return operations;
    }
}
⚠️ Naive. Each pass scans the array twice; in the worst case the number of rounds is on the order of distinct values, but values can shrink slowly — not linear in n alone (e.g. many small steps). Fine for understanding the process.

✅ Optimised (distinct positives)

Collect distinct positive integers in a Set; the answer is set.size().

✅ Solution.java — optimised 
import java.util.*;

class Solution {
    public int minimumOperations(int[] nums) {
        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            if (num > 0) {
                set.add(num);
            }
        }

        return set.size();
    }
}
✅ Key insight. Each operation removes one “layer” of the smallest remaining positive value; layers correspond one-to-one with distinct positive numbers → O(n) time and O(n) extra space for the set.

Complexity

⚠️ Naive

Worst-case many rounds depending on value distribution; each round is O(n) scan — not ideal for large constraints.

✅ Optimised

  • Time: O(n) — one pass to fill the set.
  • Space: O(n) — at most one entry per distinct positive value.
