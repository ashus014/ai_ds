CS229 Lecture 1

Machine Learning: An Introduction

01. Why Machine Learning? The Big Picture

Professor Andrew Ng opened the lecture with a sweeping vision: AI is the new electricity. Just as electricity transformed every major industry ~100 years ago, machine learning is poised to reshape healthcare, transportation, law, education, and virtually every other domain — not just technology companies.

He emphasized that the demand for ML skills is growing faster than supply, and this asymmetry creates a unique opportunity. Many industries — logistics, manufacturing, healthcare — are desperately seeking people who understand these tools, yet very few practitioners exist outside traditional tech.

Key analogy: Ng compared getting into ML today to working on the Internet 20 years ago — an early mover advantage with enormous long-term upside.

03. Prerequisites

CS229 is a mathematically rigorous course. Students should arrive with:

  • Probability & Statistics — random variables, expected value, variance
  • Linear Algebra — matrices, vectors, matrix multiplication
  • Basic CS fundamentals — Big-O notation, data structures
  • Python — basic familiarity recommended

04. What Is Machine Learning?

"A field of study that gives computers the ability to learn without being explicitly programmed."

— Arthur Samuel (1959)

"A program is said to learn from experience E with respect to task T and performance measure P, if its performance improves with experience E."

— Tom Mitchell (1997)

Arthur Samuel's checkers-playing program eventually surpassed Samuel himself — one of the earliest examples of a computer outperforming its programmer.

05. The Five Core Topics of CS229

  • Supervised Learning: Learning X → Y mappings with labeled examples (regression, classification)
  • ML Strategy & Learning Theory: Systematic diagnosis and improvement of ML systems
  • Deep Learning: Neural networks, backpropagation, foundations for advanced work
  • Unsupervised Learning: Finding structure in unlabeled data (clustering, dimensionality reduction, ICA)
  • Reinforcement Learning: Learning through reward signals (robotics, game-playing, logistics)

06. Supervised Learning — Deep Dive

Core idea: You have input–output pairs (X, Y) and learn a function that maps new X to correct Y.

Regression: Predicting continuous values (e.g., house prices from square footage).

Classification: Predicting discrete labels (e.g., tumor malignancy: benign vs malignant).

High-dimensional features: Algorithms like SVM can operate in infinite-dimensional spaces via the "kernel trick".

Real-world demo: ALVINN (1989) — neural network trained to steer an autonomous car.

Regression Classification SVM ALVINN

07. Unsupervised Learning — Deep Dive

In unsupervised learning, you have only inputs X — no labels. Discover hidden structure yourself.

Clustering (K-Means): Google News (topic grouping), genomic analysis, social networks, market segmentation.

Cocktail Party Problem / ICA: Separate overlapping voices from mixed recordings without labels.

Word Embeddings: Learn analogies like "man:woman :: king:queen" from raw text.

08. Reinforcement Learning — Deep Dive

No labeled (X, Y) pairs. The agent interacts with an environment and receives reward signals.

Helicopter example: Let the helicopter try different control actions. Reward when it flies steadily; penalize crashes. Over time, the RL algorithm learns optimal policy.

Dog-training analogy: Reward desired behaviors, discourage unwanted ones. The algorithm learns the optimal behavior pattern without explicit rules.

Applications: AlphaGo, Atari games, robotic locomotion, logistics optimization.

09. ML Strategy — The "Hidden" Skill

One of CS229's most distinctive offerings: systematic engineering principles for ML. Two teams using the same algorithm can have vastly different outcomes based on strategy.

Common pitfall: Spending 6 months on an approach that was never going to work.

Ng's approach: Writing "Machine Learning Yearning" to codify these systematic principles.

Topics covered: When to collect more data, which architecture to choose, interpreting training/test error gaps, leading ML teams efficiently.

10. CS229 vs. CS229a vs. CS230

CS229

Most mathematical. Broadest ML coverage. Rigorous proofs and theory. Heavy workload.

CS229a

Flipped classroom, more applied. Less math, more programming. Good stepping stone. Capped at 100 students.

CS230

Deep learning focus. Narrower scope but goes deep on neural networks. More practical in DL domain.

Taking Multiple

Recommended: Little overlap. CS229 + CS230 + probability, optimization, RL for true expertise.

The Meaning Behind the Math

Ng closed by noting that beyond career success, machine learning offers a rare opportunity to meaningfully improve the world — better healthcare, personalized education, stronger democratic systems.

"With these tools, you actually have the power to do that."
